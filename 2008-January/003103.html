Patch below.

After expanding exponentially what started out as a minor hack
to add Garmin GPS 18-5Hz support to gpsd-2.36 has fixed a major bug
dual pps monitor threads, fixed a few minor bugs and added generic 
nmea 1Hz square wave PPS device support.

A 1Hz square wave pulses high/low in a 1000ms cycle
changing at 0ms and 500ms. The problem was detecting if the pulse
was going high or low at 0ms.

Michael St. Laurent has kindly tested this patch for me and reported that
it worked correctly with his Furuno Electric GPSClock (GH-79L4) even when
gpsd was built with --disable-gpsclock. Which removed the custom code
for the GPSClock and relied on the generic NMEA driver. The default value
for the leading_edge variable is 0 the code detected that his GPS clock
required 1 and flipped its state.

I've tested this code against my Garmin GPS 18-5Hz and found it to work
very well at 34800bps. I'm getting a very stable 5 pulses per second with
less than 0.1ms variance. The reason I did this patch is the GPS 18-5Hz
does not do PPS it does 5PPS. Doh!

Unfortunately the GPSD list does not allow attachments so it makes it hard
to version any posted patches. This is patch version gpsd-2.36.jad-6.0 
which I hope is the final patch. If I find any bugs I will post a new patch
with the string gpsd-2.36.jad-6x To make web searches easier.

I've posted this patch so that anyone wanting to use a GPS 18-5Hz or a 1Hz
square wave nmea device can find it online. 

The dual pps_monitor thread bug only shows up when GPSD switches drivers
while running. It manifests its self by gpsd failing to write to SHM(0) &
SHM(1) and writing to SHM(2) & SHM(3) instead. If you do not wish to use
this patch to fix this bug check out serial.c in svn 4567

This patch is my own product it is not the work of the GPSD developers so
if it breaks anything blame me not them!
