Hello,

I have a Garmin GPS 18 LVC that I use with ntpd.  I would like to use gpsd 
with ntpd as that way I can have easy access to the position data and such, 
however I am unable to get ntpd to read the GPS time and PPS from gpsd. 
Once in a while, when restarting ntpd, it will read the GPS time once, then 
never read it again.  Most of the time though, it will not read either at all.

I have used gpsd with ntpd in the past successfully with the same setup, 
although I had to configure gpsd with the '--disable-reconfigure' option to 
get it to successfully read the PPS signal then.

The only two differences between then and now is I recently extended the GPS 
unit's cable by about 50 feet with some cat5, so that I can put it directly 
on the peak of the roof for maximum exposure.  I also recompiled the kernel 
with the timer frequency set to 100Hz, instead of Mandriva's default 250Hz.

I know the GPS device works fine, as I can start 'cgps' and view the 
position data and the time/date.  I'm currently using the shmpps driver that 
reads the PPS signal just fine.  When I configure ntpd's built-in GPS_NMEA 
driver, it will read the GPS time just fine.  And my FreeBSD machine has no 
problem getting the GPS time and PPS via the GPS_NMEA driver.

I've even gone as far as configuring gpsd with '--enable-gpsd-user=root', 
also no dice.

I'm using gpsd 2.36, ntpd 4.2.4p4, Mandriva 2007.0, kernel 2.6.17-16mdv, 
Garmin GPS 18 LVC 3.20 firmware.  Everything is the same as mentioned above, 
except for the kernel recompile, the cable extension and gpsd 2.36.  I did 
revert back to 2.35 and 2.34 and neither work.  I was using 2.34 and 2.35 
before successfully.

Thanks for any help!
