q_len,

This is a bug that has cropped up in the last few versions of gpsd. You 
    will need to run configure with --disable-sirfII and recompile. That 
will remove Sirf support from the binary, it will use NEMA mode only and 
that will work fine with the RGM series of GPS's. I hope to work with 
the issue this week and figure out when the bug started happening. There 
was a time that gpsd did work correctly with sirf binary mode.

It is possible to return the RGM-2000 back to NEMA mode but it takes 
some work, just keep using sirfmon and pressing n. You many need to do 
that as many as 8-10 times. Then it will be in NEMA mode, but perhaps at 
9600 bps instead of 4800. That is actually OK. Once you get the unit to 
go back into NEMA mode you can use sirfmon to reset the bps too but 
actually 9600 is a bit better then 4800 so I'd leave it at 9600.

Cheers,

David
