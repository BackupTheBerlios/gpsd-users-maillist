Lance Fetters wrote:
> The 'status' member of the gps_data_t structure returned by gps_poll is
> always (at least in my case) set to zero (i.e. no fix), even when my GPS
> receiver has a fix. The debug messages of gpsd show that the status is
> being detected properly (0 with no fix, 1 with). Has anyone else noticed
> this? This is not the proper behaviour, is it?

I investigated the problem a bit further, and found that the only
messages libgps (gps_poll) receives are of the type "GPSD,O=" and
"GPSD,Y="; it never receives "GPSD,S=".

The same is true when telnetting directly to the daemon and setting
watcher mode. However, if I specifically request the status ('s' query),
it returns the proper value.

Does gpsd/libgps not automatically return the status on a status change?
