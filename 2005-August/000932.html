The 'status' member of the gps_data_t structure returned by gps_poll is
always (at least in my case) set to zero (i.e. no fix), even when my GPS
receiver has a fix. The debug messages of gpsd show that the status is
being detected properly (0 with no fix, 1 with). Has anyone else noticed
this? This is not the proper behaviour, is it?

I tested this with svn revision 3070, using my BT i.Trek M3 receiver in
both NMEA and Sirf modes. After noticing the problem with my own
software, I tested again using a simplified version of gpslogger.c by
Tilman Koschnick (posted on the list around January). The code that I
used is at the end of this message.

As a side note, I noticed gps_poll acted differently depending on what
protocol I was using. With NMEA, gps_poll would not return until my
receiver had a fix (even though the function would still return
status=0); with Sirf, it would return data with or without a GPS fix.
