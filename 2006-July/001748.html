On Tue, 11 Jul 2006, Andrew Kezys wrote:

> Well, there's two things I need to do with gpsd. I need to tell my gps
> receiver to only give me GPGSV sentences at a rate of 1/sec. I know
> this is achievable through a certain NMEA command.  I'm guessing that
> I can tell the GPS to do this if I connect to gpsd through Python,
> however, I'm assuming that the gpspipe is only one-way? That is, I can
> get info from the receiver, but cannot send it back?

You can not sent commands to your GPS once gpsd is running with the
GPS.  Either you need to send you config commands to the GPS before
you start gpsd or you need to edit the initialization strings that
gpsd sends to your GPS.

> And if one can only connect to the serial port once, what is the
> usefulness of gpspipe being able to pipe the data to a serial port?

Some people use gpspipe to output NMEA data on a serial port for use
by other devices that only accept NMEA from a serial port.  Others use
the serial port as a way around virtual machine restrictions.  Neither
applies to you.

> So, overall, how does one go about connecting to gpsd through python?
> Do we use tcp/ip sockets, or something like pyserial to interface w/
> gpspipe?

The program gpspipe is intended for use from the command line.  If you
are going to use Python you should use gps.py to connect to gpsd.  Look
at the program gpsprof for a good example of how to do this.  In all
cases the communication with the GPS device is one way.
