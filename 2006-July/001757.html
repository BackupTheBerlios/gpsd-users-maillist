Andrew Kezys <akezys at gmail.com>:
> I was wondering if anyone could enlighten me as to where/how I can
> configure gpsd, or add some code, in order to send a string to the GPS
> receiver. I want to send a few sentences while I have the chance, in
> order to have only GPGSV sentences coming in at 1/sec. If there is any
> way of doing this by rewriting/adding some code to gpsd, or, even
> better, some kind of configuration utility or script, it would be
> greatly appreciated.

In theory, we could do this by changing the initializer in the NMEA driver.
But that would change the behavior for all gpsd users.  Not a good
idea, as most of them care more about time-and-location data than the
satellite picture :-).

I think you're making a use case for a 'gpspoke' utility specifically 
intended to ship control strings to a running GPS.  Such a thing shouldn't
be too difficult to write.  It would use the privileged control port,
and require a small hook in gpsd.c.

Are you a C programmer?  I would take a patch  to implement such a thing.
