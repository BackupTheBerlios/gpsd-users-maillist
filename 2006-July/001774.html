I find this paragraph a bit misleading as written:

    The GPS daemon (gpsd) attaches to the terminal special file and
    initializes the GPS receiver. Initialization typically sets the serial
    port speed and the receiver protocol (e.g. NMEA or binary). Once
    initialized, the gpsd continuously listens to the terminal special
    file for messages from the receiver. As messages are received, they
    are parsed and the position, navigation, and timing data is extracted.

I'd suggest eplacing it with this:

    The GPS daemon (gpsd) attaches to the terminal special file and
    automatically synchronizes itself to the type and baud rate of the 
    GPS device. Once synchronization is achieved, gpsd continuously 
    listens to the terminal special file for messages from the receiver. 
    As messages are received, they are parsed and the position, navigation, 
    and timing data is extracted.
