Warning: newbie alert! :-)

I am running a Fedora13 system using GPSD 2.39. Contrary to what I have 
seen elsewhere, 2.39 uses the OLD interface format. The gps receiver is 
a bu-353.

I have several programs attaching to gpsd and getting output. In fact, I 
have actually made the FireFox gpsd interface work. Which brings me to 
my question.

The man page for gpsd defines the "o" output as having a "tag", defined as:

     "A tag identifying the last sentence received. For NMEA devices 
this is just
       the NMEA sentence name; the talker-ID portion may be useful for
        distinguishing among results produced by different NMEA talkers 
in the same
        wire."

On my system, gpsd is sending back "o" messages with a "tag" of "GND". 
Given that this is supposed to be an NMEA sentence name, two things are 
in question. 1)Is the bu-353 an NMEA device, and 2) what NMEA sentence 
is being mapped?

I have run gpsd in debug mode and looked at the message from the gps 
unit. A typical message looks like

gpsd: <= GPS (binary) /dev/ttyUSB0: 
$GPGGA,145232,3221.3012,N,09532.6418,W,1,06,1.40,156.62,M,-25.144,M,,*79\x0d\x0a$GPRMC,145232,A,3221.3012,N,09532.6418,W,1.3853,112.526,250111,,*31\x0d\x0a$GPGSA,A,3,04,09,02,05,10,17,,,,,,,2.4,1.4,1.7*3A

The output sent to the client as a result of this NMEA message from the 
gps was

gpsd: => client(0): GPSD,O=MID2 1295967152.000 0.005 32.355019734 
-95.544030260 156.621 21.000 39.482 112.5259 0.713 0.197 ? 23.89 ? 3


I looked at the NMEA definition here:
     http://www.gpsinformation.org/dale/nmea.htm
and found no NMEA sentences that approach "GND". How does GPSD create 
the GND response which is sent back to the client?

A similar question arises when examining the xgps application. I gets a 
mix of "GND" and "MID2" messages back from gpsd. And, again, I can find 
no information that describes the NMEA sentence(s) that map to a MID2 
response.

Thanks in advance.
