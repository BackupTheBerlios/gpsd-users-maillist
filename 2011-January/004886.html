In my case, in the gpsd-2.95 directory, during the ./configure step prior to running
"make", I saw this (recovered from config.log):

<snip>
configure:18164: found /usr/bin/xsltproc
configure:18176: result: yes
configure:18188: checking whether xsltproc knows about docbook xsl
configure:18213: result: no
configure:18221: checking for xmlto
configure:18249: result: no
configure:18263: WARNING: Neither xsltproc nor xmlto works: I will not build man pages.
configure:18265: WARNING: To build man pages, install xsltproc/xsltlib or xmlto and
run autogen.sh again.
<end snip>

I went into yast software mangement, put xmlto into the search field. Selected the
rpm and then probably repeated the step for xsltproc.  It asked if I wanted some
co-requisite packages and I allowed it.

I then ran ./configure again, problem was solved and pages were built during "make".

Based upon your description, it is possible that it is complaining about one of the
python related xml packages.  Again, since I first chose to install
gpsd
gpsd-clients
gpsd-devel
via yast and *then* manually install gpsd-2.95 after that, yast would have already
taken care of any requisites that those packages needed, and that likely satisfied
what ./configure for the 2.95 level may have been looking for with the exception of
the above described xmlto/xsltproc.
