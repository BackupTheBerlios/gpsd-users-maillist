> 
> So, I'm asking if it is possible to disable gpsd without uninstalling
> it (I'm not an expert of udev).

There are several things you can do. First, use "locate gpsd.rules" to
find the gpsd rules.

* If I understand the gpsd rpm spec file correctly, you might be able
  to comment out or delete everything in gpsd.rules, and that rule set
  won't be over-written the next time you update.

* Remove them. That will get them out of udev's way. However, the next
  time a gpsd package comes down the tube, it will replace that file.
  See if there is a way to peg gpsd to its current version in yum.
