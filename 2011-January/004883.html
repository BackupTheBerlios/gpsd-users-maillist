I recently did this on an openSuSE 10.1 and a 10.3 system (surrounding releases for
10.2).

In my case, I had already installed via yast

gpsd
gpsd-clients
gpsd-devel

rpms from the openSuSE repositories. When I grabbed the higher level gpsd 2.95 tar
ball from the gpsd.berlios.de download site, I noted that the "make install" step
will by default put the new "gpsd" module in /usr/local/sbin/gpsd, and the suse rpm
package installed via yast puts it in /usr/sbin/gspd.  Since "/usr/local/sbin/"
follows "/usr/sbin" in root's default $PATH, I renamed the rpm-provided
/usr/sbin/gpsd and moved the /usr/local/sbin/gpsd to /usr/sbin/gpsd. The other
client files from the build went to /usr/local/bin and that precedes /usr/bin/ in
root's $PATH where the suse rpm put its versions of them.

After doing the "make install" for the higher level gpsd, and fixing module
locations, don't forget to run "ldconfig".

When I did the ./configure step in the gpsd 2.95 directory prior to doing the "make",
it did issue a warning about not being able to locate something it was seeking
relating to building the man pages.  I cannot recall what it was looking for, but I
went back into yast and searched/installed it, and then reran ./configure prior to
the "make". So for me, the man pages were built ok.
