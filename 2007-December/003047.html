> I tried it (with gpsd 2.34).

   That's encouraging.  I tested successfully with the libgps.so from 2.36.

> 1. no-jitter - clicked the option but not sure what to look for. But 
> the "Speed" and "Bearing" fields are not blanked between updates.

   Yep, I think that's it.  One can see the effect when using at least
some receivers in NMEA mode, where between updates some of the fields
keep getting blanked as you describe--receivers which give an atomic PVT
report will not show any difference.  In terms of user interface, the
user knows whether no-jitter is on or off by looking at the toggle
status of the relevant menu option, and obviously, by receiving a
satisfactory display without "blinking" fields (the only exception to
this would be if moving at a steady slow speed--the client has a min.
speed threshold below which the GPS unit will appear as stationary.)

   For reference, what the no-jitter option does is send a "j1" command
to the daemon, either when clicked on, or when first connecting if it
was already on. Conversely, it sends "j0" when clicked off.

   Note that if you are streaming raw data (Raw Data -> Format -> GPSD)
the no-jitter option has no effect on it.  This is by design, but not
sure whether I made the correct decision--comments one way or the other
are welcome (btw, it works that way because the raw data stream is in
reality a completely independent connection to the gpsd server).

> 2. copy position as text and KML. Both work. Nice. (Paste "Copy Position" 
> directly into the input field of Google Maps. Save "Copy as KML" to 
> somefile.KML and open in Google Earth)

   Good to know it works.  When using "Copy As KML", you should be able

that is certainly an option).  Try for example, selecting the "Temporary
Places" folder in the GEarth side pane, then press Ctrl+V (or Paste from
either the context or the edit menu).
