Michael St. Laurent <mikes at hartwellcorp.com>:
> I hacked the code of the current release to ignore the trailing edge and
> it's ugly (code wise) but does work.  If a configuration option were
> available that would allow you to specify which edge should be used it
> would be great.

One of the design objectives of the project is not to have configuration
options -- I consider it part of my job to kick, scream, and bitch whenever
anyone proposes one.  The reason is simple: we're trying to make gpsd 
totally self-configuring, zero-administration.

The way to handle this case would be to automatically switch to ignoring the
trailing edge whenever the daemon sees something in the GPSClock's output
that identifies this particular device.  Is there some proprietary sentence
it emits that we can recognize?  Alternatively, is there some probe we can
send it that would elicit a recognizable device type ID?
