Diego,
If you're updating build/run instructions for qtgpsc in non-standard locations 
then I should mention run-time which I forgot. With gpsd in /opt then allow 
qtgpsc to find libgps.so using for example this command
$LD_LIBRARY_PATH=/opt/gpsd/lib qtgpsc

I realize my posting here about Mapper is a misfit but perhaps the project 
accepts patches and perhaps someone wants to help by providing the patch to 
allow Mapper to work with gpsd.

The link to Mapper is
http://wiki.openstreetmap.org/index.php/User:Onion/Mapper
(case sensitive)

To build it on Linux i386 do what the page says under "Sources", "Other 
software" and "Checkout and Building". The only problem I encountered on 
Fedora 8 is having to make a symlink for libgstapp-0.10.so. I also 
tried "Using OSM data" (POIs) but it is not required to run the application.

I can provide some screenshots of it on Fedora 8 if interested but I have no 
place to host them.
Claus
