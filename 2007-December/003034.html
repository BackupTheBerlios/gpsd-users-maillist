After realizing I needed kernel module serial_cs, got gpsd to work on my T60 
laptop using the pcmcia slot and gpsd worked fine.

So now I am trying to get gpsd to see this device as a USB device on a 
different machine using a SanDisk 12-in-1 reader. Except when plugging in the 
BC-337 nothing happens. I am fairly sure all the needed USB kernel modules 
are loaded, etc as doing a lsusb gives me;

Bus 002 Device 005: ID 22b8:2a62 Motorola PCS E815 GSM Phone (AT)
Bus 002 Device 004: ID 22b8:0850 Motorola PCS
Bus 002 Device 003: ID 046d:c01e Logitech, Inc. MX518 Optical Mouse
Bus 002 Device 002: ID 046d:08a2 Logitech, Inc. Labtec WebCam Pro
Bus 002 Device 001: ID 0000:0000
Bus 001 Device 002: ID 0781:8919 SanDisk Corp.
Bus 001 Device 001: ID 0000:0000

and those all work fine.

Googling about it seems a /dev/ttyUSB(some number) should be created which 
doesn't happen. On the T60+pcmcia I was using /dev/ttyS0 and catting that 
device would output NEMA sentances.

So going with that on this box and using "gpsd -N -n -D 2 /dev/ttyS0" gives;

gpsd: launching (Version 2.34)
gpsd: listening on port 2947
gpsd: successfully connected to the DBUS system bus
gpsd: running with effective group ID 0
gpsd: running with effective user ID 0
gpsd: opening GPS data source at '/dev/ttyS0'
gpsd: speed 9600, 8N1
gpsd: garmin_gps not active.
gpsd: speed 38400, 8N1
gpsd: speed 19200, 8N1
gpsd: speed 2400, 8N1
gpsd: speed 4800, 8N1
gpsd: speed 9600, 8N1

But I never see any sentances. Using any thing other than /dev/ttyS0 gives 
errors along the lines of "gpsd: => GPS: @X?*67\x0d FAILED".

So what am I missing? Is there another kernel USB driver that I need?
