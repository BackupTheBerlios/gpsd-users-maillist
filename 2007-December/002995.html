On Dec 11, 2007 11:16 AM, Eric S. Raymond <esr at thyrsus.com> wrote:
> Chris Kuethe <chris.kuethe at gmail.com>:
> > something like that. point gpsd at a file full of raw gps capture, and
> > out comes gpsd-format messages. at least two of us have our uses for
> > this.
>
> Given that gpsfake does this reliably now even on OpenBSD, why add complexity
> to the C code to do it in a different way?  I'll be hard to persuade on this
> one.

because
a) using gpsfake means lugging around python which i don't want to do
on smaller systems
b) gpsfake doesn't really do this anyway - all the real work is still
done by the daemon, and using a python wrapper adds complexity anyway.
c) it's slower than reading the file and just spewing to stdout.

> Why is NG involved here?  Is it just as a mechanism for switch output formats.

I see NG as generally a richer language for both input and output.
It's unclear to me how we'd be able to reset a gps for raw measurement
output and then forward those measurements with our current protocol.
