This is a workaround for a broken kernel driver.  It will work on some
machines but has many problems.  It is very speed dependent and if lock
is every lost there is no way to recover.

> (I don't intend to interfere with whatever decision you made, I just want
> to know if the matter was resolved.)

There are 2 Garmin binary transports.  One sues <DLE>..<DLE><ETX>, and
one uses packets.  The packet type only works on USB garmins.  The DLE
works on both serial Garmins and USB Garmins with the garmin kernel
driver.

The packet transport is broken in the linux kernel for slower CPUs or
when kernel latency occurs.  So I have added a handler for the <DLE>
transport.  That works in SVN now for serial Garmins but not quite yet
for USB Garmins.  I just have a few more things to work out and then
that will also be in SVN.  Just need some quality time to finish it up,
something I am finding hard to scavenge at the moment.
