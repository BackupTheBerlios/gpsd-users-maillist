On Wed, 18 Oct 2006, Bryan Kardisco wrote:

> I have a 'DeLorme Earthmate GPS LT-20' and am using GPSD.  Have no problem
> starting it up just do the standard

Good, we would like a success report when you are done.  The format
is near the bottom of the hardware page here:

	http://gpsd.berlios.de/hardware.html


> 'gpsd - p /dev/tts/USB0' and all is well ; however, I am trying to get some
> dataout in the form of STDOUT so I can kick it to an external text file.

I recommend you start it this way, unless you are concerned about power
usage:
	gpsd -n /dev/tts/USB0

> Now I've done this through telnet no problem -- connected to the local host
> and have it just going 'p'  -  However, doing this from a shell script
> exiting the telnet session isn't very clean and I was hoping for a more
> robust way to do this.  Is it possible to get this data out in another form
> -- all I need is the lat / long and pull it out once every 2 minutes or so.

Try this:
	gpspipe -r
