I recently purchased a BU-303 GPS USB receiver, and am using gpsd to run 
it. Initially everything worked, I was able to run xgps and get expected 
results. However, now when I run xgps the status quickly changes to 
UNKNOWN and the message "no data arriving appears". I have tried a 
variety of sollutions, from reinstalling gpsd, to trying different USB 
ports, but get the same negative results.

The command stty -F /dev/ttyUSB0 ispeed 4800 && cat </dev/ttyUSB0 
returns the expected data stream, so the device is accesible and on. 
When I run gpsd I get the following message:  "gpsd: can't run with 
neither control socket nor devices", however, if I run gpsd -f 
/dev/ttyUSB0 it appears to work. If I telnet to port 2947 and issue the 
"wr" command the expected data stream no longer appears, and again if I 
use xgps I get the results mentioned above.

I've contacted USGlobalSat, the producer of the BU-303, but they have 
not been able to provide an answer.  Any ideas?

Thanks
