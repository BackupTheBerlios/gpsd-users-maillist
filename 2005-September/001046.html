Having noticed a few odd points with the revision that I had been using
(rev 3130), I decided to try the latest revision of gpsd (3200 at the
time). However, this revision crashes upon execution. I attempted to
debug it with gdb, but it even causes gdb to crash (segfault on ctrl-c).
Has anyone else had problems? I am using it on a Zaurus C3100 with a BT GPS.

On another note... some of the odd points that I noticed with rev 3130:

- for the 'X' command, the documentation states that the response will
be either 'X=0' (no gps found) or 'X=1'. However, when my gps unit is
not turned on, I get 'X=?' instead of the expected 'X=0'; is this
behaviour correct?

- in watcher mode, when the gps is off, gpsd returns 'O=?' every cycle.
I would assume that gpsd would return a 'O' line only when there is
useful data (unless an 'O' command was specifically requested). I did
not notice this in some of the earlier revisions of gpsd that I had
used. Is this normal?
