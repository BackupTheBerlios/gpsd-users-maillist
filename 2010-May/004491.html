The gpsd  documentation ( http://gpsd.berlios.de/gpsd.html ) would suggest that NMEA formatted GPS data over TCP/IP can be specified as  a data source:


TCP feed
A URI with the prefix "tcp://", followed by a hostname, a colon, and a port number. The daemon will open a socket to the indicated address and port and read data packets from it, which will be interpreted as though they had been issued by a serial device. Example: tcp://data.aishub.net:4006.

When I try to use gpsd in this manner, it fails, e.g.,:

$ gpsd -N -n -D5 tcp://127.0.0.1:10110
gpsd: launching (Version 2.94)
gpsd: opening IPv4 socket
gpsd: opening IPv6 socket
gpsd: listening on port gpsd
gpsd: NTPD shmat(65539,0,0) succeeded, segment 2
gpsd: NTPD shmat(65540,0,0) succeeded, segment 3
gpsd: running with effective group ID 502
gpsd: running with effective user ID 502
gpsd: opening read-only GPS data source type 0 and at 'tcp://127.0.0.1:10110'
gpsd: device open failed: No such file or directory - retrying read-only
gpsd: read-only device open failed: No such file or directory
gpsd: GPS device tcp://127.0.0.1:10110 nonexistent or can't be read



I'm fairly confident the source works because I've used it with other software clients (MacENC, NavMonPC). From the source end I can't detect any attempt to connect to the specified socket. 

Is this a working feature, or am I misinterpreting the documentation?

Rick
