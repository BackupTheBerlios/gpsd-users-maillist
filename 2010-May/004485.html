A Quarta, 19 de Maio de 2010 08:09:46 Martin Magnusson escreveu:
> Hi!
> 
> I have problems getting gpsd to read data from a bluetooth connection
> with a Trimble ProXT GPS. I have set up rfcomm and I can cat
> /dev/rfcomm0 and see the NMEA messages go by. However, I don't see
> anything from gpsd. I started gpsd by running "gpsd /dev/rfcomm0", but
> neither gpspipe nor xgps gives me any readings.

It should work. I usually kill gpsd first in order to avoid udev screw-ups.

$ sudo killall gpsd
$ sudo gpsd -N -b /dev/rfcomm0

(my GPS doesn't like gpsd hence the '-b'; '-N' is to run on foreground, just 
to know gpsd is alive)
