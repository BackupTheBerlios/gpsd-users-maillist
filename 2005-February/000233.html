The link to the Rockwell protocol description in gpsd.xml:
	http://www.gpsinformation.net/dale/nmea.htm
seems to be dead.

The zodiac.c file has code in it that makes me think this is at least
some subset of the protocol spoken by the PLGR (Portable Lightweight GPS
Receiver) made for the DoD by Rockwell and documented in ICD-GPS-153

The PLGR protocol used two non-IEEE floating point formats.  One is DEC,
as seen on VAXen.  The other is called CAPS.  There's a standard
precision (32 bit) and an extended precision (48 bit).

I bring this up because I don't see any code to deal with these floating
point number formats in gpsd, and I have an interest in supporting the
PLGR with GPSD.  If there's anyone else that would like to be able to
use GPSD with a PLGR then it might be worth my effort to implement a
PLGR-to-NMEA translator.
