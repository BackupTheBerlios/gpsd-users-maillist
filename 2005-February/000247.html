On Tue, 2005-02-08 at 16:23, rlynch at bway.net wrote:
> I'm looking for a utility that would be able to create a fake serial
> device, and then feed arbitrary (made-up) GPS position data to gpsd.  I
> figured it might be worth a shot to check the list, and see if any
> developers had a testing tool that would do something like this.
> 
> It's pretty simple--it just need to feed a user-specified position to gpsd
> in such a way that the daemon thinks it's attached to a GPS receiver that
> actually is at that position.

I think another useful tool would be a client that connects to gpsd,
possibly running on a different machine than gpsd, reconstructs an NMEA
stream from what it reads from gpsd, and sends well-formed NMEA messages
out a serial port.  This would enable using a linux laptop/embedded box
to provide GPS data to multiple devices that expect NMEA data over
serial.
