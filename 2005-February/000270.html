Hi,

I'm having trouble linking to the libgps from the RPM, while linking to
one I built from source is fine.

I believe the problem is that the %install section of the spec file is
not using "make install".  By using cp ./lib/xxx instead of "make
install", libtool is unable to do its magic.

As a result when linking my application, libtool is translating "-lgps"
into "/usr/lib/.libs/libgps.so" which does not exist.

Here's a thread that discusses a similar issue:
http://www.redhat.com/archives/rpm-list/2002-October/msg00107.html
