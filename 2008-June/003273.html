So I've been playing around with my Pharos iGPS-500 and gpsd and have
been having trouble receiving 'P' data via a PHP socket. It's the
strangest thing. gpsd is running fine, and I connect via fsocketopen
and fwrite($handle,'P\n'). I always receive a GPSD,P=? (the equivalent
of, I forget). However, if I manually telnet to localhost and write
'P', the first I get a 'GPSD,P=?' response, but the second time I get
my correct lat and long. Now if I leave the telnet connection open and
try my PHP script again, it works great, and always returns the exact
same lat and long that I last retrieved using telnet. But as soon as I
disconnect my telnet session to localhost, my PHP script stops
retrieving any data. However, this only happens with P (position) and
A (altitude) data. O works immediately in both a telnet environment
and by using the PHP script and connecting via fsocketopen.

I hope that all wasn't too convoluted to make sense.
