Various stories exist online of people compiling GPSD under Cygwin.  A 
useful idea since providing GPS data to the network would be nice from any 
platform.  Tinkering with 2.2 and now 2.3 I've had ever decreasing problems, 
but still cannot get it to work. (Note: I have GPSD compiling and running 
wonderfully under Slackware for Kismet, Cygwin has been far more of an 
adventure.  My goal is a working Kismet setup under windows, the last piece 
is the GPS data, everything else works).

To get get 2.3 to compile, you need:

libgpsd_core.c - Add #include <sys/socket.h>
gps.c - Add #include <stdio.h>

If you add those, 2.3 compiles cleanly.  Running it is where the problems 
start:

$ ./gpsd -p /dev/ttyS2 -D99
gpsd: launching (Version 2.3)
gpsd: listening on port 2947
gpsd: Selecting Generic NMEA driver...
gpsd: closed GPS
gpsd: opening GPS data source at /dev/ttyS2
gpsd: setting speed 4800, 8 bits, no parity
gpsd: gpsd_activate: opened GPS (4)
gpsd: => client(128): GPSD,X=1
gpsd: Client on 128 has vanished.
<last line repeats 4 times with an ever increasing Client #>

telnet to port 2947 connects until a keypress then the daemon closes with a 
"Bad File Descriptor" error.

cat </dev/ttyS2  shows normal looking NMEA GPRMC sentances coming through.

Tried with GPSgate simulator software and with Garmin Etrex Legend.  Both 
work on the same machine with different apps.  Cygwin version 1.5.11-1 from 
Cygwin.com, full install

Any insights?
