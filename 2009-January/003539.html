Eric Klein <klein.eric at gmail.com>:
> I'm developing some software at the hobby level using gpsd, and have
> been attempting to get gpsfake to work properly. It seems no matter
> what NMEA log I throw at it, it only ever sends gpsd the first NMEA
> log line. I have tried using logs downloaded from a couple of gps
> hobbyists, and also purchased an Earthmate LT-40. The LT-40 works like
> a champ with gpsd, with xgps and gpsdrive having no trouble following
> the satellites and track. However, even the NMEA file I generated
> using gpspipe is only passing the first NMEA log line to gpsd in
> gpsfake. Am I just missing something? I'll be happy to post to the
> list or privately any diagnostic info you need.

We recently became aware of this bug ourselves.  It is not a gpsd problem,
but the result of some kind of gnarly interaction between Python, the
kernel, and even the GCC version. What we know about it is described here: 

	http://gpsd.berlios.de/upstream-bugs.html#gpsfake

The workaround is to drop back to Python 2.4 for running gpsfake.  The
fix is in somebody else's code :-).  We are continuing to investigate;
just today we turned up evidence of a Linux kernel bug in pty handling
that we'll almost certainly file a bug on.
