Mike Pilkington <MPilkington at lgc.com>:
> Thanks for the feedback, Eric.  Here's something interesting...a few days ago I found at least a workaround for the issue of running the BU353 GPS receiver in VMware.  Ran across the following link, which at the end describes the use of the 'stty' command.  I believe it sets the baud rate for the serial port, though I'm not too familiar with this command.

That's what it does, all right.  A classic Unix command.

> At any rate, after running the following command, it works in VMware:
> 
> stty -F /dev/ttyUSB0 ispeed 4800 && cat </dev/ttyUSB0
> 
> Sometimes I get an error and have to kill it and re-run it.  I've found it always works on the second try for some reason.  Once I see output from this command, I Ctrl-C it and then run 'gpsd /dev/ttyUSB0', start up Kismet, and everything is good!
> 
> Strange!

Yeah.  Sounds to me lilke VMWare has unhappiness somewhere in its serial-device
emulation that mokes baud-rate changes failure-prone.  That would explain both
gpsd's failure to hunt properly and flukey behavior from stty.
