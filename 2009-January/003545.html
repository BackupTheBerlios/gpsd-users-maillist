Try what Eric said first (invoke "python24" instead of just "python").

I don't know about Python specifically, but in general, I find that
most packages that use GNU configure don't mind being installed
into /usr/local or even /home/username (or, if you're insane and/or
fighting partition space, /var) for testing purposes.  Don't worry
about using dpkg or RPM to install the old version - just download
the source tarball and say "./configure --prefix=/usr/local" or
whatever directory you want.  It will make the subdirectories (bin,
lib, man) if they don't already exist.  You *will* make mistakes with
$PATH and invoke the wrong program a couple of times; it's
probably not a good idea for production systems, but for testing
it usually works fine.
