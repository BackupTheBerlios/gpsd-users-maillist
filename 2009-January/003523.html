I'm trying to set up gpsd (2.37 on Fedora 8) with an old Trimble
SVeeSix-CM3 using the TSIP protocol (which runs at 9600 8-odd-1).  The
problem is that gpsd starts at 4800 8-none-1 and never seems to try
anything else.

Running with debugging, gpsd is getting garbage characters (at the wrong
baud and parity), trying to parse them (which fails), and never tries to
shift baud rates.

There doesn't appear to be a way to override the autobaud or
autodetection.  Auto-everything is nice, but when it fails and there's
no way to override, the program isn't much use.
