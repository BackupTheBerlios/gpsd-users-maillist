On 12/09/2010 12:09, Joe Martine, N5USR wrote:
> I feel like I must be missing something obvious, but perhaps gpsd doesn't
> work the way I was expecting...
>
> I have a utility I'm writing in Python that simply needs to grab the
> current position fix every so often when it logs something.  I don't need
> a continuous flow of live data from the GPS, just an update every N
> seconds - which is normally going to be 10 or more seconds apart.
>
> Working with the gps.py module, it seems all I can get is a constant flow
> of data, and if I don't run poll() fast enough the data is buffered and I
> wind up getting old data instead of current.  At least I believe that is
> what happens, the "time" property only increments by one second with each
> poll(), regardless how often I call it.
>
> Is there a way to request a single fix on demand, rather than a constant
> stream?  Some of what I read in the docs and mailing list archives makes
> me think I can, but all the examples I've found are for streaming data.

Joe,

This is my experience too. In my C++ wrapper I have an 
option to do a flush-poll, where I repeatedly poll until I 
can't poll any more, then use the last sample.

This requires disabling the automatic function callbacks and 
manually invoking callbacks from within the wrapper, but 
that's fine as I only use synchronous polls.

If there's a more direct way to flush libgps's serial buffer 
I'd love to hear it too!

Tom
