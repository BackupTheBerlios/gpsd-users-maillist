On Sat, 3 Nov 2007, Keith E. Brandt, M.D. wrote:
> I am trying to install gpsd from the ports distribution of FreeBSD6
> (system was completely updated approximately a month ago). It looks
> like it is using gpsd-2.32. When I try to build it, make returns a
> series of cascading errors which start with:
>
> gpxlogger.c:8:18: glib.h: No such file or directory

I have glib.h on my system and the comments indicate it's part of GLib.
The docs say "GLib is a library containing many useful C routines for
things such as trees, hashes, lists, and strings.  It is a useful
general-purpose C library used by projects such as GTK+, GIMP, and
GNOME."

I don't know if FreeBSD does this, but on Linux it's fairly common for
libraries to have a main package (like "libfoo") and a development
package (like "libfoo-dev").  The main package contains only the
libraries, and is useful for precompiled software that just needs to
link against the libraries.  The development package contains the
headers that are needed to compile software that uses the library from
source code.  You may need to install the GLib development package in
order to get gpxlogger.c to compile.

> gpxlogger.c:9:23: dbus/dbus.h: No such file or directory

Same story with dbus.h .  The docs say "D-Bus is a message bus, used
for sending messages between applications.  Conceptually, it fits
somewhere in between raw sockets and CORBA in terms of complexity."
There is a dbus package, plus a dbus-glib package that says "This
package provides the GLib-based shared library for applications
using the GLib interface to D-Bus."  You may need to install the DBus
and DBus-GLib development packages in order to get gpxlogger.c to
compile.

Another option, at least for dbus.h, would be to tell the build process
to not look for DBus at all.  It's been a while since I built something
on FreeBSD, but if you still get to run "configure" yourself, you might
say

configure --disable-dbus

and it shouldn't even try to build the stuff that needs DBus.  Note that
the --disable-dbus option os from the configure shipped with gpsd-2.34 ;
I don't know if it was in gpsd-2.32 or not.

I hope this helps!
