I have made some changes to the webgps client I wrote.  I am happy with it
for my purposes so am putting it out there now for anyone else to play
with.  It is essentially the analogue of xgps but generates some html and
svg.  As it runs continuously in the background by updating the data once
per second this version also keeps a record of the satellite positions so
it will plot the path of the birds while they are in view.

Webgps can be found at: http://www.wireless.org.au/~jhecker/gpsd/

The program generates two files gpsd.svg and gpsd.html which are to go
into a path that can be accessed by a web server.  I am using minihttpd on
my little gpsd box to serve the page up on my private network.  The pages
render in Konqueror, Firefox and IE with the Adobe SVG plugin.  Opera
complained for some reason, I didn't look hard for the reason.

/* To use call: webgps [server] [port] [path] */
/* server: the gpsd server you are connecting to */
/* port: the port number of the gpsd server */
/* path: the destination path for gpsd.svg/html */
