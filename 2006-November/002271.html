Well, it may be too late for this, but I've got some slight  
improvements and bug fixes for gps.py.

I am attaching a diff file and my complete changed file. I couldn't  
easily get a diff -Naur to work right since there seems to be some  
drift in the whitespace that added all sorts of incorrect changes. So  
its just a diff -w orig new

The main improvement was that it passes the self structure as part of  
the polling data so you can have structured access to the data. I've  
been using this but its possible that there are things still broken  
with it. I am having some problems getting altitude and speed  
correctly but it may be do to some other things... Haven't had a  
chance to fully test those. So you may not want to add this yet to  
the new release but someone may want to check it out when they get a  
chance.

The bug fixes were in the return of some data functions. One had the  
order of the arguments to time.strftime backwards and the other spelt  
gmtime as gtime...
