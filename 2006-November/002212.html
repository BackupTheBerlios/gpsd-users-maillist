> Chris, I think the keyword here is "wine" under Linux, and AFAIR .net
> doesn't run under wine (yet). If wine can provide a virtual comport on
> the emulated side, that connects to a TCP socket on the Linux side,
> then gpsd should work for windows based programs running under wine.
> But I think this is something the wine developers should take care of,
> as it is on the wine side of things. GPSD uses the standard TCP socket
> interface, so it should probably not be to hard to do for the
> codewizards at winehq.

But if we(the gpsd) developers would be able to create a virtual /dev/ttySxx 
which is connected to gpsd then this would have more benefits for us. For one 
think a wine should be able to connect to this virtual port without blocking 
the real gps receiver. For another example; applications like Navigator(for 
Linux)/Directions(for Linux) could share a gps receiver too without blocking 
the gps-receiver.
