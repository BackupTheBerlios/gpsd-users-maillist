On 11/7/06, Johnson, Charles F <charles.f.johnson at intel.com> wrote:
> Folks - I'm trying to understand if there already exists RDS-TMC support
> in Linux
> for any of the receivers that may exist. (RoyalTek RTM-2000 for
> example.)
> I don't understand things well enough to know if this is a "gpsd"
> question, an application question or something else.

Short answer: we don't support that.

Long answer: A quick google suggests to me that RDS-TMC is some kind
of routing metadata protocol... Yup. Radio Data System - Traffic
Message Channel. It looks like traffic and road status is transmitted
on an FM subcarrier, and your in-car navigation system handles that as
well as realtime positions from your GPS.

I don't see any posted protocol documentation for the Royaltek TMC
protocol, nor do I have one of these devices, so I can't write any
sort of hook for it.

It's also not really something gpsd would know what to do with. Gpsd
is a protocol converter and redistributor. It makes it easy to have
kismet, roadnav, gpsdrive, etc. all reading from the same receiver at
the same time, but it doesn't do any navigation for you. In the
context of RDS-TMC, I'd think that if we did ever support this
protocol, it would just be in a retransmit mode, and we let your
applications decide if they need TMC data. The aforementioned roadnav
and gpsdrive would be likely consumers of TMC.
