I've put a new web page on the project site that aims to name and shame
GPS vendors that commit truly egregiious blunders.  It's at 

http://gpsd.berlios.de/hall-of-shame.html

Your contributions are welcome.  If you're describing a firmware bug, it's 
best if you can identify the firmware version or range of versions it applies
to.

Way back when, there was a really stupid low-end GPS chipset that only
issued correct checksums for packets when it had a fix.  That deserves
to be in the Hall Of Shame.  Does anyone remember the vendor and model
number?

(I think it might have been from Holux.)
