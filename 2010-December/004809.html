Carlos Rivera <crivera at envitel.com>:
>     I needed an application to log gps data so that it can be retrieved
> later to determine the trajectory of the machine. Since I could
> not find anything appropiate I wrote a small tool in python to do the job. 
> I have posted it at: http://superkaos.org/gps_logger/
> It has the following features:
> 
>     * It uses GNU autotools.
> 
>     * By default It runs as a daemon. It contains an upstart script
>       that gets installed in ${sysconfdir}/init  (i.e. typically
>       /etc/init).
> 
>     * The logs are stored at ${localstatedir}/log/gps (i.e. 
>       typically /var/log/gps).
> 
>     * Old logs are deleted after a certain amount of time controlled
>       by a variable in the code.
> 
>     * The data is indexed by utc timestamp.
> 
>     * There is also a small library called gps_logger_query.py to access
>       the logs and it has a function to connect to gpsd directly and 
>       stream the data back to the caller, the idea is to use this library 
>       to access the stuff via http, both logs and real time data, but 
>       that's not done yet.
> 
> Anyway, I just wanted to let you know that the code is freely
> available for anyone who might be interested.

It is possible, if the code and documentation of this tool is up to 
our project standards, that we might want to carry it in the gpsd 
distribution as one of the clients.  Is that a possibility that
interests you?
