I just got a new udev package on Ubuntu 10.10, udev 162-2.2. Knowing
that udev changes out from under us from time to time, I did some
checking. After removing the socket and shutting down the gpsd daemon:

* There seems to be a problem running the gpsd udev code. It fires up a
  daemon, the socket created, and my client can connect to the daemon.
  But my client never sees any message indicating that there is a GPS
  receiver. Similarly with xgps.

* I can run the hotplug script manually,

  ./gpsd.hotplug add /dev/ttyUSB0

  and the daemon runs, the client gets the receiver message almost
  immediately, and then gets fixes, etc. as usual. This makes me wonder
  if in the first case, gpsd isn't opening the device file correctly.
  The script is getting the correct device file as an argument.

* I found that the build process ("make install")
  installs /etc/default/gpsd, and this (START_DAEMON="true") tells the
  startup code to run gpsd as a daemon. Launching gpsd at boot as a
  daemon works correctly. So it is possible this has been going on for
  a while.

  (Since the idea is to run gpsd from udev, perhaps this line should be
  "false"?)

My receiver is a BU-353, and it appears as a pl2303 serial device.
