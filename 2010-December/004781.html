Hi

On the latest Fedora release, GPSD seems to be installed (and launched) by
default.

BT747 (http://www.bT747.org), an opensource program to manage MTK Chipset
based GPS Loggers, requires bidirectional communication with the logger.
As GPSD already occupies the serial port (in a to the user 'invisible'
manner), BT747 can not connect to the port.

The only solutions I see are:
- Somehow 'kill' gpsd upon BT747 launch;
- Uninstall gpsd;
- Integrate an interface to gps in BT747.

I've had a quick look at the libgpsd.c code to see if it was somewhat
portable to java, but this does not seem to be the case.
It looks like some kind of virtual device is used - is it possible to
connect to that?
BT747 needs to be able to write/read the serial data to/from the GPS device.
 Some sample usage of libgpsd would be usefull.  It surely can be wrapped in
a native library for java.

The highlevel protocol does not allow bidirectional communication
apparently.

What are the GPSD expert suggestions?
