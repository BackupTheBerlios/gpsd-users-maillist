On Wed, 29 Dec 2010 06:13:22 -0500
Eric Raymond <esr at thyrsus.com> wrote:

> Charles Curley <charlescurley at charlescurley.com>:
> > BTW, something else to add to the udev debugging list: On
> > Ubuntu/Debian, check for /etc/default/gpsd having
> > 
> > START_DAEMON="false"
> > 
> > If that is "true" the startup code will launch gpsd at boot, which
> > will be confusing if you aren't expecting it.
> 
> bzed, why do we ship a configuration file with START_DAEMON="true"?

gpsd doesn't. I found out late last night that in addition to the gpsd
HEAD, I also had a debian gpsd package installed. /etc/default/gpsd
came from that package. So if you are working with gpsd HEAD, and find
that file, remove the package.
