> I just got a new udev package on Ubuntu 10.10, udev 162-2.2. Knowing
> that udev changes out from under us from time to time, I did some
> checking. After removing the socket and shutting down the gpsd daemon:
> 
> * There seems to be a problem running the gpsd udev code. It fires up
> a daemon, the socket created, and my client can connect to the daemon.
>   But my client never sees any message indicating that there is a GPS
>   receiver. Similarly with xgps.

I found the problem. Somehow I managed to have the gpsd debian package
installed. That's where the file /etc/default/gpsd comes from, and I
missed that clue earlier on. I did an "apt-get purge gpsd" and this
file went away, and this problem seems to have gone away.

Now to find out why and how I got gpsd installed.
