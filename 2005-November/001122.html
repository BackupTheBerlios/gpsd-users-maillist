> Devices are not really too different.  Are you using a Mac with a  
> serial port or a usb adapter?  If you have a Prolific based usb  
> adapter the correct device would be /dev/cu.usbserial1.  There will  
> also be a /dev/tty.usbserial1 but I have found that the cu. device  
> always works while the tty. device sometimes does not.  If you have a 
> Keyspan adapter it will look something like /dev/cu.USA191b1P1.1.  Do 
> a 'ls /dev/ |grep cu.' to find out what your serial port is called.   
> If you try to run gpsd and it hangs you might want to check
> permissions.

I've got a PL+SirfIII usb (BU-353) which loads onto /dev/cu.usbserial,
thanks for the hint. Sirfmon hasn't picked it up, but it is working fine
in NMEA mode so I'm not worried.

gpsd 2.30 builds and runs with just "./autogen.sh && make". cool.


three issues:

1) "stty -F /dev/cu.usbserial ispeed 4800" as in step 1 of INSTALL gives
an error (not encouraging as a step 1). OSX 10.3 doesn't know about -F.
(port is set at 38400) Installing minicom and setting baud to 4800
shows good NMEA.

2) `ln -s /dev/cu.usbserial /dev/gps` doesn't survive a reboot. Gpsd
starts without /dev/gps there but silently exits the first time you
connect to the port (even at -D5). Anyone know a Mac work-around for
this?

3) "./configure --help" reports this:
-----=====-----
Installation directories:
  --prefix=PREFIX         install architecture-independent files in PREFIX
                          [/usr/local]
  --exec-prefix=EPREFIX   install architecture-dependent files in EPREFIX
                          [PREFIX]

By default, `make install' will install all the files in
`/usr/local/bin', `/usr/local/lib' etc.  You can specify
an installation prefix other than `/usr/local' using `--prefix',
for instance `--prefix=$HOME'.

For better control, use the options below.
=====-----=====

So I didn't use --prefix because I want it in /usr/local/.
But `make install` installed to /usr/, which wasn't something I wanted
to mess with on someone else's Mac. grumble. !@^#$%??!!!


otherwise working great so far. (using with gpsdrive via Fink)

It would sure be nice if someone put together a Fink port for gpsd. :)
