On Sat, 26 Nov 2005, Rafal Mantiuk wrote:

> But when I try to get a response from the GPS:
>
> > stty -F /dev/ttyUSB0 ispeed 4800; cat </dev/ttyUSB0

ispeed?  You want to set the input speed and not the output speed?
ispeed sets the input speed only, ospeed sets the output speed only, dns
speed sets them both the same.  I suggest your try this instead:

stty speed 4800 < /dev/ttyUSB0

Not all Unix allow the -F but all allow the redirection.  That said, all
USB is at 1.5/10/400mbps so setting the speed to 4800 baud is odd.  In
many cases the speed on a USB tty is just a placebo to keep apps happy.
I would just leave the speed as it is on power up.  So first I would
power cycle the system and try simply  this:

cat /dev/ttyUSB0

Best case setting the speed in the USB channel is a no-op.  Worst case
you set the speed in the pl2303 at a mismatch to the Magellan core.  It
would be odd that the Magellan defaults to such a slow speed on such a
fast link.

Other thing to check, are you sure you have the right USB port for the
GPS?  Did you see /dev/ttyUSB0 in the dmesg or syslogs?  Send the output
of this so we can see the driver got the unit:

	cat /proc/bus/usb/devices
