As the GPSD/NTP server I am putting together is destined to live on a Windows 
Only network (outside of the occasional Knoppix boot) I am going to be bereft 
of the highly useful and pretty xgps client (cgps doesn't have pretty 
pictures).

So I have butchered the xgps code to spit out html and svg instead to see 
what's going on via any SVG capable browser (IE with the Adobe SVG plugin, 
Mozilla/Firefox and Konqueror - I have only tested it with the latter).  You 
can see an example and get the very raw code at:

http://www.wireless.org.au/~jhecker/gpsd/

The tarball has a new Makefile.am so just pop it into your gpsd v2.34 tree and 
rebuild from scratch to get webgps built too.  I was thinking it could be 
turned into a daemon or some sort of cgi-bin.  Unfortunately SVG can't be 
inlined (as far as I know) in HTML so a cgi-bin may not be entirely viable.  
I'd appreciate any feedback and ideas.
