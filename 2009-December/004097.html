We're making good progress in wrapping up the 2.90 release. This is
a fast update on three issues:

1. FP-related port bugs

The only remaining pre-shpping issue is that mledford and gdt havee
found some port bugs on i386 and amd64 that appear to have something
to do with variations in word width and flosting-point arithmetic on
these platforms.  The port bugs manifest as a inconsistent values for
time, altitude, and course-error estimates. Michael Ledford is taking
the lead in trying to fix these.

2. More robust code in gps.py

I've just ripped out the use of the rather buggy sockfile feature in gps.py,
replacing it with my own buffering logic.  This may enable people on
OpenBSD to run the regression tests without the hangs and other
problems they've been reporting. If you're one of those people, please
test and report.

3. Packaging issue: Python 2.6 vs. Python 2.4+ & simplejson

mledford also contributed the key bit of information that the widely
available 'simplejson' third-party module is plug-compatible with the
Python 2.6 library json module. I have modified the gps.py code so
it will import simplejson if json is not available; our deb and
RPM packagers will need to tackle building packages that will 
accept either combination.
