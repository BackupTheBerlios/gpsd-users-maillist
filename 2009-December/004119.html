If I use gps_send() to send a request for date/time (using "d\n"), a 
subsequent gps_poll() never returns.  Even if the file descriptor for 
the gps device indicates that data is ready (via select() function), the 
gps_poll() locks up rather than returning a response.  If I just use 
gps_stream(), select() and gps_poll(), the gps_poll() call always 
returns when select says data is available.  As soon as I inject a 
gps_send() into the loop, the gps_poll() freezes up.
