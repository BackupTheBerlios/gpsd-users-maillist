The only thing I was waiting on before shipping was for Chris Kuethe to
conform that the OpenBSD build is working.  As a bonus, Andre Naujoks's
ASCII dump of RTCM data has allowed me to confirm that the RTCM2 analyzer
works correctly.

This is a 48-hour imminent-release notice.  Please run all the tests you can.
I will be accepting emergency bug fixes and documentation improvements only.

There are still known problems with float stability on odd
architectures.  They manifest mainly as missing speed and
error-estimate fields.  I'm not enough of a numerical analyst to know
how to fix them.

This release may be the one that gets tagged 3.0.  I've been reserving
that bump for when the core elements of the JSON protocol looked completely
stable.  I think we've reached that point.
