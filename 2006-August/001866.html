Neal Probert <nprobert at probestar.com>:
> Can one gpsd server use another gpsd server for DGPS information?

Unfortunately, the answer is that I do not know if this works.  In
theory I think it could -- I've been moving the architecture towards
handling RTCM data streams as smoothly as GPS ones, and there's an
incompletely implemented RTCM_SERVICE option that makes gpsd serve
RTCM packets on port 2101.

But lacking the peculiar setup needed to test this code, I don't actually
know how well the DGPS features work.  We have regression tests that 
verify RTCM packet decoding, but that's about it.

I apologize that you get to be a guinea-pig.  The good news is that
the hard, nasty part (RTCM decoding) is done. The bad news is that you
may have to do some testing and minor code tweaking  to make gpsd
support the particular information flow you want.

I'll take clean patches to support it.
