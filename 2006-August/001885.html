When ntpd is properly configed for gpsd you should not get refclock errors.

> Hope I have provided enough info....

Please post your ntp.conf so we can check it.  Also, are you trying
to use PPS?  Here is what mine looks like using PPS:

driftfile /etc/ntp/drift
multicastclient                 # listen on default 224.0.1.1
broadcastdelay  0.008
server 204.17.205.1
server 204.17.205.2

# for gpsd
server 127.127.28.0 minpoll 4 maxpoll 4
fudge 127.127.28.0 time1 -0.500  refid GPS

# for PPS and gpsd
server 127.127.28.1 minpoll 4 maxpoll 4 prefer
fudge 127.127.28.1 time1 0.0003 refid GPS1

logconfig =syncevents +peerevents +sysevents +allclock
