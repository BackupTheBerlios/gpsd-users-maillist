An older version of gpsdecode seems to return a different (correct)
position for an AIVDM type 3 packet than version 3.4 that is packaged for
Ubuntu 12.
The old version returns an immediately useable lat and lon, the new version
returns totally different numbers (see below).
If I run the same sentence through http://rl.se/aivdm it returns the
correct info, same as the old gpsdecode version.

Same thing happens with type 1 packets.

The old version when run as *gpsdecode -V *returns : SVN ID: $Id:
gpsdecode.c 6965 2010-01-30 14:19:42Z esr $
This older version also returns "Moored" for status and the 3.4 version of
gpsdecode returns 5 for status...

gpsdecode -V
SVN ID: $Id: gpsdecode.c 6965 2010-01-30 14:19:42Z esr $

echo '!AIVDM,1,1,,A,37PCkb5000blLHWjG88enI8T2DW:,0*78' | gpsdecode -j

{"class":"AIS","type":3,"repeat":0,"mmsi":503641000,"scaled":true,*
"status":"Moored"*,"turn":0,"speed":0.0,"accuracy":true,"lon":*151.2666*
,"lat":*-23.8352*
,"course":3545,"heading":292,"second":18,"maneuver":0,"raim":true,"radio":168852}


The new version when run as *gpsdecode -V* returns : gpsdecode revision 3.4

gpsdecode -V
gpsdecode revision 3.4

echo '!AIVDM,1,1,,A,37PCkb5000blLHWjG88enI8T2DW:,0*78' | gpsdecode -j

{"class":"AIS","type":3,"repeat":0,"mmsi":503641000,"scaled":false,*
"status":5*,"turn":0,"speed":0,"accuracy":true,"lon":*90759955*,"lat":*
-14301150*,
"course":3545,"heading":292,"second":18,"maneuver":0,"raim":true,"radio":168852}

the examples below were run on different systems, one with the old version
and one with the new version.
*Where have I gone wrong?  I can't imagine that gpsdecode 3.4 has a bug
this bad that hasn't been noticed by others.*
