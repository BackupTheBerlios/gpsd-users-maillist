I stumbled over the new geolocation api support in firefox where it says firefox now supports gpsd (https://developer.mozilla.org/en/using_geolocation). So grabbed a bt dongle and a bt gps...

It seems gpsd does not find bt devices bound to a rfcomm serial port emulation; or does the bt recipe need an update?

I managed to connect gpsd and my holux GPSlim236 via bluetooth. I followed the instructions in the recipe for bt GPS receivers (http://gpsd.berlios.de/bt.html) and realized it does not work that way. 

I got that far that I could see the NMEA messages with a

cat /dev/rfcomm0

but then it recommends to hack this line into /etc/init.d/gpsd

GPS_DEV="/dev/rfcomm0"

Obviously this does not work anymore this way. Also the telnet protocol seems to having changed as the client doc (http://gpsd.berlios.de/client-howto.html) now recommends issuing a

?WATCH={"enable":true,"json":true} 

which only returns an empty devices array (congrats on choosing JSON btw):

{"class":"DEVICES","devices":[]}
{"class":"WATCH","enable":true,"json":true,"nmea":false,"raw":0,"scaled":false,"timing":false}

After some googleing I found a reference hinting at /etc/default/gpsd which in turn needs to be altered with a (says the comment in this file)

dpkg-reconfigure gpsd

it looks now like this:

# Default settings for gpsd.
# Please do not edit this file directly - use `dpkg-reconfigure gpsd' to
# change the options.
START_DAEMON="true"
GPSD_OPTIONS=""
DEVICES="/dev/rfcomm0"
USBAUTO="true"
GPSD_SOCKET="/var/run/gpsd.sock"

The above mentioned watch statement produces now

{"class":"DEVICE","path":"/dev/rfcomm0","activated":1278025002.59,"flags":1,"driver":"Generic NMEA","native":0,"bps":57600,"parity":"N","stopbits":1,"cycle":1.00}
{"class":"DEVICE","path":"/dev/rfcomm0","activated":1278025003.00,"flags":1,"driver":"SiRF binary","subtype":"GSW3.1.1_3.1.00.07-C23B1.03a ","native":1,"bps":57600,"parity":"N","stopbits":1,"cycle":1.00}

plus other stuff and some positional data, so seems to be fixed that far...
:D

Two questions arising now:

1) I am new to gpsd but I could contribute that information as an update to the above mentioned bt recipe if only anybody could tell me how or point me at whom to ask.

2) I tried that test service from mozilla (http://www.mozilla.com/de/firefox/geolocation/) but it seems it does not use gpsd's data or is not working at the moment at all (before installing gpsd firefox said I am in lower Austria which is not the case at the moment... ;-)) 
Does anybody know how to get firefox's gpsd support working or are there test pages out there or is there a contact at mozilla to ask?
