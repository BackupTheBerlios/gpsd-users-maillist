Following up on an earlier post a couple of months ago, I'm trying to connect gpsd to an NMEA message server program using tcp. (I know the server program works in practice because I've used in conjunction with another client.) When I try the following (server runs on same processor as gpsd) on OS X 10.6 (Snow Leopard):

$gpsd -n -N -D2 tcp://127.0.0.1:10110

I get the following diagnostic messages:

gpsd: launching (Version 2.94)
gpsd: listening on port gpsd
gpsd: running with effective group ID 502
gpsd: running with effective user ID 502
gpsd: opening read-only GPS data source type 0 and at 'tcp://127.0.0.1:10110'
gpsd: device open failed: No such file or directory - retrying read-only
gpsd: read-only device open failed: No such file or directory
gpsd: GPS device tcp://127.0.0.1:10110 nonexistent or can't be read


Looking at the source, it appears to be failing in gpsd_open (serial.c) when it calls open(<filename>,<mode>) with the tcp URL.

My knowledge of C and low level I/O is minimal, so can anybody confirm that a tcp URL (e.g., tcp://127.0.0.1:10110)  should work as a filename argument for open, and this works on OS X?

Thanks for any info or guidance,
