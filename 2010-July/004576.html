Hi,

 i had the silly idea of trying to (try) to crosscompile the latest gpsd
(2.94) on Arm. Specifically i'm trying to build it for a Moxa UC-7110.
According to cpuinfo, this device is a 
Processor       : ARM922T rev 1 (v4l)
BogoMIPS        : 75.98

 Moxa delivers a toolchain with the device with limited libraries and a 
really old gcc:
gcc version 2.95.3 20010315 (release)(ColdFire patches - 20010318 from
http://fiddes.net/coldfire/)(uClinux XIP and shared lib patches from
http://www.snapgear.com/)

 Problems started with trying to run configure with this toolchain,
which i tried with setting the following flags
declare -x CC="/usr/local/arm-elf/bin/gcc"
declare -x CPP="/usr/local/arm-elf/bin/gcc -E"
declare -x LDFLAGS=" -Wl,-elf2flt"

 With those options, this gcc produces no a.out, but a file called -r
instead. Looking through the configure Script i believe this happens
where configure tries to figure out using -o or not, and i believe in
case the first doesn't work it tries again with the wrong variable...
anyway, after making a small adjustment there (Changing line 3815 to
(eval "$ac_link") 2>&5 ) at least make configure run through.

 However, make doesn't get very far...  anyway: I'm trying to figure out
if getting this running is even remotely possible :) or if i should save
my time and look for different hardware. Suggestions?
