Rusty, 

You tell the error is this, right?
    import gps.fake
ImportError: No module named gps.fake

I think its clear; python can't find gps.py or fake.py. 
You need to tell him where it is.
I dont know if gpsd needs python 2.6 or 2.7, 
but I dont think it has anything to do with it. 

You fake.py is here

/usr/local/lib/python2.6/site-packages/gps/fake.py

so you just need to add that path to python,
again PYTHONPATH controls this. You could try reading a bit
of python:

http://docs.python.org/release/2.6/using/cmdline.html#envvar-PYTHONPATH

So, after setting PYTHONPATH=/usr/local/lib/python2.6/site-packages/gps
do this:

python -c "import sys; print sys.path"

To check the path python knows for its modules. This path must
include gps directory for gps.py.

Hope it helps.
