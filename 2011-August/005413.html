In previous mail, I wrote:
>I have attempted to fix the scons issues with rpath pointed out on IRC but 
>neither bzed nor mike_t have been around to verify the fix.

I just had a very productive IRC session with mike_t.  The known issues with 
rpath and installation in non-default locations have been fixed.  Also,
the leapsecond cache file is no longer clobbered if you try to rebuild 
without Internet access.

The absolute-path problem in the RPM spec has not yet been fixed.  mrd
sent a minimal patch so he can build using it; we'll solve in full after
release.

The locale problem in the C++ binding has not been fixed.  I would
accept a patch for this.

3.0 will ship sometime around noon Thursday, barring some major 
last-minute panic.  I'm still accepting emergency fixes, but not
expecting any.
