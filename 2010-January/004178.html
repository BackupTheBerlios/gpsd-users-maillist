some off-topic (wrt gpsd) comments re. your program. Perhaps they can help
you save some time by not having to reinvent some wheels, and are
interesting to the rest of the list in some way.


 - why make users geo-register the NOAA charts manually? The free RNCs
you download from NOAA in BSB format can be very easily converted directly
into a geo-registered GeoTIFF (or whatever) with the `gdalwarp` program
from www.gdal.org. The `gdalinfo` program will give you access to the
already-embedded x,y to lat/lon control points (or use libgdal directly
in your app and make it all automagically work for the user on the fly).
  http://www.gdal.org/frmt_various.html#BSB
some more tips here:
  https://sourceforge.net/apps/mediawiki/gpsdrive/index.php?title=Marine_Howto
(GpsDrive wiki)


 - JPEG is probably not as good as PNG or compressed TIFF for those
charts as they are usually based on < 16 color palette and large blocks
of constant color. The JPEG image will be both a bigger file and of less
quality.


 - If you would like to support electronic nautical charts (vectorized ENCs) in S-57 format (also a free download from NOAA; but encrypted
in the UK, other govts [ppht]) you can load it (well the unencrypted USA
data anyway) with the OGR library from www.gdal.org.
  http://www.gdal.org/ogr/drv_s57.html
  http://grass.osgeo.org/wiki/S-57_data


 - A very nice S-52 rendering library for that S-57 formatted data can
be found here:
  http://openev.cvs.sourceforge.net/openev/contrib/S52/

(libS52 stands out as one of the most impressive FOSS mostly-solo efforts
I've seen. a huge effort & incredible dedication by Sylvain Duclos... go
man go.)


 - there is some FOSS nav software called "OpenCpn" for plotting S-57 ENC
and BSB charts based on an older version of Sylvain's libS52 library.
links:
  http://opencpn.org/
  https://sourceforge.net/projects/opencpn/
  http://www.cruisersforum.com/forums/f134/


hopefully in all of that there is some room for collaboration.
