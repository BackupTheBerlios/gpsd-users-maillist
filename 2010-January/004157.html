As far as I know, there are two parts to A-GPS.... First there is the:
"which tower am I on, and where is it located?" Somehow cellphones can
do this, and provide a location estimate well before the GPS can have
a signal lock.

Secondly there is some information, which is transmitted from the GPS
satelites at a low baud rate. If all goes well (an ideal signal), this
information takes up to 2 minutes to transmit/recieve. This is
neccesary to obtain a position fix. On the other hand, you can just as
well download this information from the internet. It's the same all
over the world (but changes fast enough that 3 hours later it's
invalid enough for my GPS not to trust it anymore)

Provided we'd know where to get this information, and in what format
it is, we could download it with whatever internet link we happen to
have.

However sending it to the GPS unit poses another problem. There
usually simply isn't any protocol to do this.... 

(on the other hand, for example the SIRF chipset needs to be able to
be built-into a cellphone. So the cellphone needs to be able to tell
it the "assist" information. A GPS mouse need not consist of much more
than a SIRF chipset, plus a bluetooth<->serial chipset plus a
battery. So therefore it is possible that sending the right strings
over bluetooth works.)
