We track a high speed vehicle. In fact, we use the GPS to locate
position in real time. Our systems require action when at certain
locations (which we may only pass tangentially). We have found that with
a 10 Hz receiver and 38400 baud (and Linux which can have respectable
throughput for these sorts of things) the GPS information allows more
accuracy/repeatability than the operator pressing a button. In this
case, we are talking 1 meter accuracy. I should note that in addition to
the GPS we use a wheel pulse transducer. This helps between GPS fixes.
After all, with a 10 Hz receiver, the locations will be provided every
2.5 meters at 90 km/h.

You can of course buy systems (e.g., www.oxts.com) that bundle all this
(GPS + INS + wheel pulse) in to a nice package that provides NMEA data
in real time at a very high frequency. It also provides data over UDP
broadcasts for even less delay. Of course, these cost a bit more.
