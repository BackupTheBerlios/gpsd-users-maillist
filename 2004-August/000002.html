I've created a OpenEmbedded build script of the new gpsd and have been
able to test it out a little on my iPAQ 3835.  I'm using a "transplant
GPS" CF card.

The application I really want to work is "qpegps" which I will work on a
solution for once I can resolve a few things.

First I can't seem to get the old functionality of GPSD working.  When I
telnet to the port and type "a" or "p" or "s" it seems to only work once
in awhile.

If I do a "w" I see all kinds of good data.  My first guess what the the
"GPS_TIMEOUT" was too short, but that seems to make no difference.  My
next guess was the processGPRMC was clearing all the REFRESH timers, but
even if I put a return in the "else" block I still can't get reliable
data out.

The two command blocks that I eventually want to send (from qpegps) are
the following:

	SPAVHBDX for gps status, and SPWAVHBLDC for mapping.

When the first one is given I get values for SDX only.  If I keep typing
A (or P or S) I will get one every once in awhile.

Attached (if it makes it) is a block of the NMEA from my GPS device. 
The result of an R until the light indicated it had a fix, followed by
another R and then a W.  In the middle I though I did a few ASP commands
(on the change I might catch a returned A or P value) but I don't see
them.
