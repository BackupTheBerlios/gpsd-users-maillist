I'd like to report success (although it's somewhat limited) with 2.35b 
(SVN 4408) with this particular GPS bought from Conrad in Germany.

1. Model Name: NaviLock NL-209PU
2. Works: Yes (NMEA only)
3. URL: 
http://www.navilock.de/produkte/gruppen/3/Kabel_Empfaenger/61371_NL-209PU.html?show=spec
   (in German)
4. Sony CXD2951 Low Power with PL-2303 prolific
5. Interface: Serial interface using 6-pin connector with power supply 
at 4800 8 N 1. Comes with a 6-pin adaptor to USB so it's powered 
directly from USB.
6. Vendor: NaviLock
7. Version: 2.35beta (SVN 4408)
8. NMEA: v2.2
9. When running, need to use the -b option, else the device will hang 
during GPSD's probing phase and it needs to be unplugged and reinserted. 
For the "fudge" factor of NTP, I had to use a time of 0.840. There is no 
PPS signal associated with this device that I know of.

# GPSD
server 127.127.28.0 minpoll 4 maxpoll 4
fudge 127.127.28.0 time1 0.840 refid GPS

server 127.127.28.1 minpoll 4 maxpoll 4 prefer
fudge 127.127.28.1 refid GPS1

The technical information talks about 1us accuracy on the clock, but I 
can't find any specs for their interface to know if there's a 1PPS 
signal or not that can be used (except it's TTL).
