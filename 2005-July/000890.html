"Eric S. Raymond" <esr at thyrsus.com> writes:
> Amaury Jacquot <sxpert at esitcom.org>:
>> what about creating an RTCM-104 stream from the data provided by a fixed 
>> GPS ?
>
> Hm.  It might be possible.  We'd have to pick up pseudoranges from the GPS;
> I was planning to do this anyway for RINEX-dumping support.  I don't know 
> what if anything has to be done to massage that data into corrections.

If I understand the problem correctly we would need to calculate what
each of the pseudoranges should be using the satellite's ephemeris
information, the precise time and our surveyed-position.  We then
subtract what the GPS measured from what we calculated and we have our
pseudorange correction.  There are several open-source gps projects we
could use code from to do the calculations.  

One fear I have is that GPS's might tell us a peudorange but not tell
us the time that the observation was made with enough precision to be
of any use for calculated the correction.  We would need to know the
observation time down to a handful of nS.
