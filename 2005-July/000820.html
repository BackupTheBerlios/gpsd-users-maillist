Dana Rawding <dana at rawding.com>:
> I've downloaded and installed gpsd on my Linux laptop and it seems to  
> work well.  However some of the software that I use does not have the  
> ability to connect via a socket.  It will only connect to a serial  
> port (/dev/ttySx).
> 
> Is there a way to echo the output of gpsd to a 'virtual serial  
> port'?  I have search the web and I've found many applications on the  
> Windows side that performs this exact function however I have not  
> found anything under Linux.

It's possible.  There's an undocumented client in the contrib directory 
of the gpsd source distribution called gpspipe that I think could be modified
to do this eithout much trouble.

I've been reluctant to, however.  I fear that if gpsd supplies raw data,
lazy application developers will take that as a signal that they need never
convert to using libgps.  This would be bad for users, as applications that
read raw NMEA are prone to various flaky failure modes when they don't
see eaxactly what they expect.

What software are you using?  Maybe it's time to lobby the author(s).
