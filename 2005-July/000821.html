I find this a real pain. I'm using a couple of packages that really need
a 'serial port'. One is the "Directions Ltd" mapping package...

the others run under windows, so I need to give VMWare a serial port for
them to talk to...

So I currently use a little script like this:

  mkfifo /tmp/navigatorgps
  while :; do
    { echo r1; sleep 9999999; } | \
      socat - tcp4:localhost:2947 > /tmp/navigatorgps
  done


But this is really hacky. I would love gpsd to provide an (optional) raw
nmea port or psuedo tty.
