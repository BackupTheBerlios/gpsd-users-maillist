Eric S. Raymond writes:
 > gpsd has no configuration file.  It was carefully designed not to.

Well, it doesn't *need* a configuration file, but if you really want
one (e.g. because you're running gpsd from /etc/init.d and want the
configuration stored in a separate file), you can run gpsd like this:

gpsd `cat /etc/gpsd.conf`

and then put your gpsd options into /etc/gpsd.conf like this:

-p /dev/gps
-s 1200
--icbm 40.03503,-75.52005

etc.
