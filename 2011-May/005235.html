> ...
> No.  The SCons documentation explains that it ignores PATH, looking for
> commands in a fixed set of system directories, so that the build environment
> will be explicitly controlled by the recipe rather than implicitly by
> environment variables of which the user may be unaware.

That's not really comfortable, but I understand that having a fixed
set of tools makes everything more predictable. [1] describes
propagating PATH into SCons if you change your opinion on that.

>
> Our build used pkg-config without problems on my machine.  Do you have
> it installed or configured in some unusual way?

The problem is that when you have a cross environment you can't use
the systems pkg-config (because you have different libraries installed
in your environment than you have on the system; also the paths
differ). Ofc. normal compilation should be not effected by this.

To solve the cross compilation issue you would have to propagate not
only CC/..., but also pkg-config from the environment (like it's done
in [2]). Because you don't want to respect PATH, every tool used
should be probed for an environment counterpart. This way you get the
right (cross-environment) pkg-config which then tells you the right
(cross compiled) libraries to link against.
