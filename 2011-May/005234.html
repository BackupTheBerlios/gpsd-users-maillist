Michael Lippautz <michael.lippautz at gmail.com>:
> While tinkering around with latest gpsd in my cross environment
> (OpenEmbedded) I noticed that the latest git version using SCons does
> not respect PATH in any way. Although it grabs CC and friends from the
> env (and  one could set CC=PATH+CC) it misses PATH and pkg-config is
> therefore not able to determine the correct includes/libraries.
> 
> Did I miss something?

No.  The SCons documentation explains that it ignores PATH, looking for
commands in a fixed set of system directories, so that the build environment
will be explicitly controlled by the recipe rather than implicitly by
environment variables of which the user may be unaware.

Our build used pkg-config without problems on my machine.  Do you have
it installed or configured in some unusual way?  We'll work with you
to find a solution, which might involve adding another configuration
switch.
