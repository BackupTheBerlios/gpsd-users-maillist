On Mon, 21 Jun 2010, weller at andreas-weller.de wrote:

> Now I discovered it has a build in fixed port speed so when compiling
> gpsd with ./configure --enable-fixed-port-speed=57600 gpsd even manages
> to work and reconfigure the GPS receiver without -b switch.

Eric is trying to eliminate command line options, not add them.

You can short circuit the auto baud if you use stty to set the bit
rate before starting gpsd.  Like this:
	stty speed 54600 < /dev/ttyS0
	gpsd -nN /dev/ttyS0
