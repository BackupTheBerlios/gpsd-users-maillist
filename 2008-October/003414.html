Victor Alvarez Roig <victor.alvarez.roig at pildo.com>:
> It seems that the control string sent to the control socket must be an
> actually string of characters.
> Is there any way to send binary UBX messages through gpsd?

At the moment, no.  This limit is a consequence of the fact that gpsd
uses C's null-terminated strings. To fix it, we'd have to implement
some kind of encoded-binary version of send.  Are you a competent enough
C coder to do this?
