/opt/freescale/usr/local/gcc-4.2.1_uClibc-0.9.29/usr/powerpc-linux-uclibc/bin/gcc

/usr/include was getting added by ./configure because on fedora no_x was no and on my xununtu install it was yes

I just added a line to my .spec file (tells ltib how to configure) to say --without-x



if test "$no_x" = yes; then
  # Not all programs may use this symbol, but it does not hurt to define it.

cat >>confdefs.h <<\_ACEOF
#define X_DISPLAY_MISSING 1
_ACEOF

  X_CFLAGS= X_PRE_LIBS= X_LIBS= X_EXTRA_LIBS=
else
  if test -n "$x_includes"; then
    X_CFLAGS="$X_CFLAGS -I$x_includes"
  fi
