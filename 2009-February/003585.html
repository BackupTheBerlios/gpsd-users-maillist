Hiya,

I know you said 'simple', but if you can handle something a little heavier 
than a text file (or need a database for other reasons), I can recommend 
using PostgreSQL with the PostGIS extension.

PostGIS extends the database schema and adds functions to support 
geographically aware data-types (i.e. points - 2D,3D & 4D; lines; polylines 
and areas). It lets you do standard SQL queries along the lines of "SELECT 
all points within x metres of this polyline" and so on.

PostGIS also manages co-ordinate transformations between different map 
projections. i.e. you can INSERT the data into the database in WGS84 (GPS 
native), and SELECT the data back out in whatever projection you want (as 
long as it's been defined!).

The interface to PHP is via the standard postgreSQL database functions.

There is a bit of a learning curve though - especially if you haven't used 
a GIS before.
