FWIW, I've gotten the prolific driver working on a Mac (download direct
from their website) IIRC I had to reboot for it to take effect.
(driver version 1.2.1r2)

once NMEA is streaming to the /dev/cu.* gpsd (and other GPS software) see
it fine & all is beautiful.

be sure you are really in NMEA mode and not in Garmin Protocol mode if you
want to test with cat / minicom.


I never did fully figure out the set baud rate/ stty stuff mentioned in
the GPSD INSTALL file. (what is given there is not valid for the Mac [+BSD?])
see:
  http://thread.gmane.org/gmane.comp.hardware.gps.gpsd.user/2423/focus=2461

if the port speed is wrong you should see data streaming out, but it'll be
garbage characters.
