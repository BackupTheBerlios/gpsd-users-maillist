>gpsd has no concept of frames.  We looked at that for a while but there
>was no consistency in the order that different GPS send NMEA sentences.

Well, it must have some concept of it (although I made up the name 
"frame" to refer to a block of NMEA sentences with the same 
timestamp), because the code was already in there to handle it... it 
just happened to be wrong in a couple of key places.  I've patched 
those, and it works exactly as it should now.

>So gpsd just reports what it see, when it sees it.

If it really only did that, it would be no more use than a serial 
port doubler, and we could do away with ever command except R.  In 
fact, it reports all of what it has most recently seen, when a client 
asks for it. :)  That means we can use V and P and A and other cool 
commands like that.
