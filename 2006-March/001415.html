It seems the latest update to gpsd includes a change in the "make" stage
which means it no longer builds "out-of-the-
box", at least on Mac OS X (where it always built without problems before).
It seems the culprit is that it now requires 
"xmlto".  Furthermore, initial attempts to install "xmlto" fail since that
requires a command-line "getopt" supporting "--
longoptions"... and in turn that won't install because of some linker
errors.

This seems unnecessarily complicated... any chance of rolling back the
change that introduced a requirement for 
"xmlto", since it appears xmlto may not be installed by default on many
systems?
