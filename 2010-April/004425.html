Charles Curley <charlescurley at charlescurley.com>:
> And how does the client know to issue a WATCH? If a new client comes on
> line, no problem. But if a client is already using gpsd, how would it
> detect that the server (which may be another machine) has a new gps?

If a client is on line and a new (USB) GPS becomes available, gpsd will be
notified via the hotplug system, and add the device to its pool.  It
will then notice that there are clients with active WATCHes, and open
the device.  Every watching client will get a DEVICE notification.

> Also, it would be nice to know if a gps has gone away. If you only have
> one, that means the data the client is displaying is bogus.

As was pointed out in a reply, you do get a device notification on detach.
