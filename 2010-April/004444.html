> Yes. Pull git head and read about using tcp:// URLs as data sources. I know
> this feature works because I use it to collect an AIS feed from AISHub. The
> code doesn't care what's in the packets, though. If you ship it GPS data, it'll
> treat the TCP stream as a GPS source. 
> 
> In recent shipped versions we don't have tcp:// URL support yet. Tell it 
> ais:// instead, that feature used the same code and doesn't care.
> 
> We're working on accepting UDP datagrams, too. That's not in yet.
