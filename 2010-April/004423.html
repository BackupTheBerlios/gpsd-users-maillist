On Fri, 23 Apr 2010 02:00:39 -0400
Eric Raymond <esr at thyrsus.com> wrote:

> The way it's intended to work with a wired device is like this:
> 
> 1. You plug a GPS into a USB port. 
> 
> 2. The hotplug system notices that the device that just went active
> might be a GPS, and sends gpsd a message to that effect.
> 
> 3. gpsd adds the device to its pool, but does not yet open it.
> 
> 4. Later, a client app issues a WATCH.  At this time, gpsd tries
> to activate all the devices in its pool.

And how does the client know to issue a WATCH? If a new client comes on
line, no problem. But if a client is already using gpsd, how would it
detect that the server (which may be another machine) has a new gps?

Also, it would be nice to know if a gps has gone away. If you only have
one, that means the data the client is displaying is bogus.
