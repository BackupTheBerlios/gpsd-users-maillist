Håken Hveem <krbjhvee at online.no>:
> Hello, are BINR, RTCM SC 104 data protocols supported by gpsd ?

No and yes.  That is, we do support RTCM but we do not support BINR.
Yet. I know of it and a little about what it looks like, because I have
a partial translation of a Russian manual for it.  I know enough
to be pretty sure we could support it easily if we had documentation
for it.
 
> The BINR, RTCM SC 104 protocols is transmitted by the CF-4706 chip
> used in the CH-4711 GLONASS/GPS mouse:
> http://www.nvstech.ch/?page_id=230
> 
> By default the device does not report 2d fixes; the vendor
> configuration tool offers checkboxes to enable any combination of
> none, 2D fixes, 3D fixes, or both. The devices has only a very
> limited set of NMEA controls but speaks a proprietary vendor format
> called BINR with more capabilities.
> It is supported by gpsd ( NMEA data)
> 
> nvstech, that is the european vendor for Navis can be reached at:
> info at nvstech.ch
> 
> the maker of the chip can be reached at :
> navis at navis.ru

Right.  If you want it supported, here's what we need:

1. English-language docs for the protocol (that is, both the NMEA
subset and the BINR part).  A decent translation of the Russian or
Norwegian manual (as opposed to vendor-issues docs in English) would
be good enough.

2. Log files of BINR data to test with.  They need to be annotated 
with a source latitude and longitude so we can check that what we
get from the decoding is sane.

3. If the controls to change the reporting are documented BINR 
or NMEA strings, that's all we need.  If they're something 
else, we'll need you to capture them with a serial-port snooper
and tell us what they are.

The good news is that once we have these things, full support will
be at most two man-daus of work.  Maybe one day.  We've done
this dance before, a lot.

I'll send a request for protocol documentation to these addresses, but
you should assume it will fail and gather the information we need
yourself.  GPS companies tend to be run by idiots who think protocol
secrecy somehow adds value to their hardware rather than subtracting
it; they'll probably ignore me.
