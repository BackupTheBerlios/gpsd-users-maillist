I have an ET-301 SiRF-III engine board and I'm trying to use it with
GPSD 2.30.

The host platform is arm-linux-gnu (little endian) on a PXA-255 CPU.

I have found that I have to force GPSD to use /dev/ttyS2 to startup
correctly.  The host computer has /dev/ttyS{0-3} as valid serial
devices.

The device docs claim that it supports up to 57.6k baud, but GPSD will
not raise the baud rate above 4800 using the b= command.

GPSD cannot change the device to SiRF binary mode using the n=1 command.
The i command returns "GPSD,I=Generic NMEA".

At the moment, I am attempting to use an active antenna with the engine
board; it claims to support them.  Do I need to direct the device to use
the antenna in active mode using some kind of SiRF message?  Should I
expect it to autodetect the antenna type?  At any rate, the device does
not get a fix, and doesn't appear to see any satellites.

Any help ya'll can provide for hunting down these issues would be
appreciated.
