Gary,

It's been my experience that zero-config is a dream and I've never seen 
it work in the real world 100%. Certainly the current gpsd fails to auto 
config the serial speed correctly and detection of gps type for some 
gps's. Given that I prefer a smaller simpler binary that drops the auto 
config stuff in favor of a simpler binary that can be told what to do 
correctly the first time.

As I recall the UNIX way was is simpler binarys that work as opposed to 
complex binarys that work most of the time and fail sometimes. I can go 
with use stty to set the port if all of the auto-detection serial stuff 
was removed, then the binary is simpler, smaller and more reliable.

Cheers,

David
