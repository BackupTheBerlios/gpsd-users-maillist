Gary,

No, it actually will not depending upon which gps you have hooked up. 
I've found that the only way to get gpsd to work correctly is to use 
/bin/stty -F port 4800 before invoking gpsd.

It would be much better if the command line option still existed but 
alas that is gone. Eric now has one of these gps's that cause this 
issue, but there is more then once kind of gps that exhibit this problem.

> I think gpsd tries the current port speed first, so use stty to set it
> correctly to speed up start up:
> 	stty 4800 raw < /dev/ttyUSB0
> 
> I note one issue, since you are trying to force the tty to use, you
> should try this command line instead:
> 
> 	 gpsd -NnD 2 -p /dev/ttyUSB0
