Hi!

I'm a new user of gpsd and kismet.
They work fine together.

But I've got a problem with my gpsd (gps-reciver is a evermore-chip with 
pl2303-converter):
When the gps-reciver losts the gps-signal (not enough satellits in 
view), gpsd goes crazy.
The gps-reciver sends, after a "reconnect" to the gps-satellits, new 
positions to the gpsd and gpsd distributs them.
So far, so good.
But the problem is, that gpsd never changes the Mode-argument any more.
Even after unplugging the gps-reciver gpsd doesn't refresh the Mode-value.
The result is:
kismet ignores the log/lat, because it thinks there is no fix.
gpsdrive works quite well again, because it uses the raw-stream of gpsd.

Is this a problem of my gps-reciver or is it a bug in gpsd?

And is there any way to gpsd say something like this:
When there you lost the gps-signals to the satellits, calculate the 
current position each second with the last valid position, the heading 
and the speed?
