Well, the latest build from the SVN server (rev 1110) builds much nicer and 
requires no hand massaging to get compiled (thanks!).  Problems still 
persist in running it but whatever you guys have been tinkering with lately 
seems to be helping get closer.

GPSD no longer seems to crash out after a short time, and continues running, 
but cannot seem to open the device or deal with connected clients on the 
network side without crashing with a 'select : bad file descriptor' error. 
Output attached below.  Seems like 2 seperate problems.

I can cat data from that port just fine with a simulator and a Garmin eTrex 
Legend so I know that part works.

Seems to be some wierdness with the linux select() implementation dealing 
with the MS winsock function (no surprise there)

Any idea on an appropriate bounty to offer to get GPSD working on Cygwin?
