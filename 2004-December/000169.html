Yo gps-users!

Got a success report for you.

   1. The GPS model name

	Garmin GPS-18

   2. Whether or not it works

	Seems fine so far.

	I used garmin_gps Ver 0.19a and garminusb2nmea Ver 0.12

	Both from  http://sourceforge.net/projects/garmin-gps/

        I can not get Linux 2.6.9 or 2.6.10 to work on my laptop so
        I can not try newer versions of garmin_gps.

   3. If possible, an URL to a technical reference on it

	http://www.garmin.com/manuals/GPS18_TechnicalSpecification.pdf
	http://www.garmin.com/support/commProtocol.html

   4. The GPS and (if applicable) USB-to-serial chip it uses

	Dunno, could not get gpsprobe to work for me.    I keep
        getting python errors I do not understand.

   5. The interface type (RS232, USB, CF)

	USB.

   6. The vendor

	Garmin

   7. The gpsd driver type (almost always 'n')

	Yep, -n.

   8. Version of gpsd you tested with

	Version 2.5

  11. Notes on the device

        Known caveats of garminusb2nmea:
                - DOP (Dilution of Precision) information not available
                  (Garmin protocol includes EPE only)
                - DGPS information in GPGGA sentence not returned
                - speed and course over ground are calculated from the
                  north/east velocity and may not be accurate
                - magnetic variation information not available
                - Garmin 16-bit SNR scale unknown
