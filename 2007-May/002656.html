1. does the Gps device send information(message) on request(from the Application)?
 Or the Gps device is always pumping messages to the receiver.

The GPS almost always outputs data gratuitously. Under rare
circumstances you may have to poll things from the receiver, or engage
in some bi-directional protocol. But that's very rare. And you'll know
when such things are needed...

2. As far as i got to know, type of messages and their sequence is vendor dependent 
and there is no fix protocol to be followed. Nmea 0183 talks about a lot of messages 
and each vendor selects a few from that list. Is this correct?

The exact output order (as far as we can tell) is non-standard. There
are a handful of sentences that make sense and are a good idea for a
receiver to output. These are generally implemented by all vendors.
Other sentence types (namely VTG and ZDA) are not as heavily used -
these types are often either not enabled by default or are totally
unimplemented in the receiver's firmware.

3. is there any ack of the command?

the ack of the command is vendor specific..
in case of GARMIN
a). if there are no errors in the sentance it will be echoed upon receipt. if an error
is detected, the echoed PGRMI sentence contains the current defualt value. 
b). all configuration changes take effect after receipt of a valid value except baud 
rate. baud rate changes takes place on the next powercycle or an extrenal reset.
in case of FASTRAX
the command is acknowledged by the device by echo of the smae command along with the 
checksum added to it. 
.......................
Depends. Some receivers just do what you ask of them or fail silently,
other receivers send an explicit message. SiRF -> $Ack (OK), uBlox ->
$PUBX (error message), Fastrax -> $PERR (error code)

If there are any specific acknowledgements required, those would be
documented in the device's hardware and/or binary protocol manual.
Garmin seems to make the only devices that need a handshake. Most
other receivers tend to happily spew out navigation data whether or
not anyone is listening.

informs the receiver accordingly.
