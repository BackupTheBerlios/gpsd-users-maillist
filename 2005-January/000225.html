Since you asked...

I have been playing, mostly sucessfully, with the GPS 18 since getting
one for Xmas.  OK, it was a self gift, but the thought was there. :-)

The garmin-gps driver is now up to rev 0.23 and part of the linux
2.6.11-rc1 kenel.  Not had great luck with kernels past 2.6.7 myself
but I can confirm the 0.19, and 0.23 drivers work.  The 0.19 had some
problems if you used a USB mouse at the same time, 0.23 seems better
but not used it much yet.

Both driver versions will segfault gpsd afer a while.  Killing and
restarting garminusb2nmea and gpsd a few times will revive it.  Not
dug in to that yet.

The garmin-gps page on SF also has a daemon garminusb2nmea.  That daemon
can read the garmin propietary protocol and output simple NMEA on a
pipe.  The pipe can then be read by gpsd (as /dev/gps).  Not had any
luck using it with the "-d dgps.wsrcc.com" option though...

Here is a sample NMEA output from garminusb.nmea:

$GPGSV,2,1,06,04,61,082,39,17,32,189,40,02,56,175,45,09,45,226,44*71
$GPGSV,2,2,06,30,23,305,43,05,56,305,39*76
$GPGGA,194141,4404.1339,N,12118.8432,W,1,06,,1129.4,M,-20.1,M,,*6B
$GPRMC,194141,A,4404.1339,N,12118.8432,W,000.0,000.0,200105,,*0B
$GPGLL,4404.1339,N,12118.8432,W,194141,A*3A
$GPGSA,A,3,,,04,,17,,02,,,09,30,05,,,*13

A native garmin decoder for gpsd would be great.
