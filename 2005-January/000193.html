On Tue, 2005-01-11 at 12:03 -0500, John wrote:
> Is there a list of gpsd clients available somewhere? I'm looking for a 
> simple log of location data over a trip, written to a text file. I 
> suppose I could write something myself, but if something is already 
> designed that makes it easy to later import the tracks into other 
> programs (i.e., uses some standard I'm not aware of), that would be 
> ideal. Many thanks!

I've written a small program that does just that, available here:
http://www.subnetz.org/gpslogger/gpslogger.c - not very well tested,
though. To be linked against libgps from the gpsd package.

It takes a format string as a description how the logfile should look
like, and writes to stdout. There are options for time and/or distance
intervals. A format string would look like "lat: %l lon: %L time: %u" or
just "%l %L %u". The idea is to have a collection of pre-defined format
strings, and an option for each, so that '% gpslogger -F > file' creates
the right input format for program Foo. Any such format strings are
welcome.

Cheers, Til
