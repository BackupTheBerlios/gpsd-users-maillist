>
> you can use xastir with vector maps in shapefile form converted from 
> census maps (download them from xastir.tamu.edu). xastir is a gpsd 
> client.

That's probably the most straight forward solution but if you are into
some functionality overkill you could also try my real-time plugin for
GRASS GIS; although a bit clunky and based on old versions (GPSD 1.1
and GRASS 5.0), it takes care of the task it was written for nicely.

screenshot of vector coastline + gps track, stats, & 20 minute trend
arrow:
 http://bambi.otago.ac.nz/hamish/grass/realtime_gps_4.png
and with more advanced vector bubble plotting:
 http://bambi.otago.ac.nz/hamish/grass/realtime_gps6.png

info:
 http://grass.institutoecos.org.br/grass-list/arq/10779.html
 http://grass.institutoecos.org.br/grass-list/arq/9997.html


You get support for all raster/vector map formats supported by GDAL/OGR
(which is most):
http://gdal.org/
http://gdal.org/ogr/


FYI something else which doesn't cover the real-time nature of gpsd you
are probably thinking of; typical use with the GIS is downloading the
waypoints and tracks at the end of the day. This is very easy with
gpstrans + GRASS GIS 5.7 = v.in.garmin module. I'm planning on one day
updating this to talk to gpsbabel as well..


> about all it doesn't support is route-finding; aside from tmrs (which

> has very limited route finding in the code and isn't useful generally

> yet) i know of no open source program which does.

You would probably just have to write the glue to interface my realtime
GIS cludge with GRASS 5.7's vector path modules[*]: d.path, v.net.path,
&/or v.net.salesman.

see:
 http://mpa.itc.it/radim/g51/d.path.jpg
 http://grass.ibiblio.org/grass57/manuals/html57_user/d.path.html
 http://grass.ibiblio.org/grass57/manuals/html57_user/v.net.path.html
 http://grass.itc.it/grass57/manuals/html57_user/v.net.salesman.html

http://grass.ibiblio.org/grass57/manuals/html57_user/v.net.steiner.html

[*] well, there's more to it than that but my point is the hardest part
of the problem is already done by those modules + the new GRASS vector
engine.
