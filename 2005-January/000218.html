I've just discovered (or had reported to me, and confirmed) a crash 
bug that is new in 2.7.  It seems (at least on Darwin systems, 
possibly on others) that repeated querying of GPSD will cause it to 
crash.  This doesn't appear to cause problems if you just turn on a 
stream using "w" or "r".

To reproduce:
- install gpsd-2.7 (possibly on Darwin only, possibly on other BSD 
systems or all systems)
- start gpsd (I've tested it with a USB GPS connected with a poor sky 
view, but can't find a pattern for the crash as being when the GPS 
does or doesn't have a fix)
- telnet to the gpsd port
- repeatedly query gpsd (e.g. p<enter>p<enter>p<enter>...)

Unfortunately it's not reliable as to how quickly it will crash, but 
I generally find within the first couple of dozen queries (I've only 
once seen it go for more than about 50 queries - I usually use either 
"p" or "q").

If you prefer to automate the testing process, just run KisMAC or any 
comparable client software that sends repeated queries rather than 
listening to a stream.
