On Wed, 2005-01-12 at 03:44 -0500, Eric S. Raymond wrote:
> Tilman Koschnick <til at subnetz.org>:
> > It takes a format string as a description how the logfile should look
> > like, and writes to stdout. There are options for time and/or distance
> > intervals. A format string would look like "lat: %l lon: %L time: %u" or
> > just "%l %L %u". The idea is to have a collection of pre-defined format
> > strings, and an option for each, so that '% gpslogger -F > file' creates
> > the right input format for program Foo. Any such format strings are
> > welcome.
> 
> This sounds like it might not be a bad thing to have in the gpsd
> distribution.  What are you using as a timestamp format, ISO8601?

Yes, I'd like to see it included with gpsd at some point. Right now, it
needs to be polished some more, and it's lacking a man page.

As for the timestamp, I'm using the utc member of the gps_data_t
structure, so it is ISO8601 format. I will add specifiers for the single
components of the time stamp, so that people can make their own format.

Cheers, Til
