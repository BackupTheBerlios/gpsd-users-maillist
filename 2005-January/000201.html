I can't seem to get your program to work with shell redirection. It 
works if I run it from a normal command prompt with the output going to 
the screen, and I can successfully format the results, but if I ever 
try to redirect, I get an empty file.

I've tried:

gpslogger > gpsdata.log
gpslogger -f "%u:%l:%L" -d 5 -i 60 >> /home/user/gpsdata.log

and even:

script -c gpslogger gpsdata.log
script -c gpslogger /home/user/gpsdata.log

I'm not getting any permission errors, and I can write to the file 
normally (or redirect something like ls -a) but I can't get anything 
from gpslogger to go anywhere else. I've also tried piping it to tee, 
or to another program without luck.

Does anyone have some suggestions? I must be overlooking something 
awfully simple. In response to another comment, this would be a great 
function to have built into gpsd itself, where a logfile and format can 
be specified, perhaps even some kind of ring buffer/incremental 
filename system.

> I've written a small program that does just that, available here:
> http://www.subnetz.org/gpslogger/gpslogger.c - not very well tested,
> though. To be linked against libgps from the gpsd package.
>
> It takes a format string as a description how the logfile should look
> like, and writes to stdout. There are options for time and/or distance
> intervals. A format string would look like "lat: %l lon: %L time: %u" 
> or
> just "%l %L %u". The idea is to have a collection of pre-defined format
> strings, and an option for each, so that '% gpslogger -F > file' 
> creates
> the right input format for program Foo. Any such format strings are
> welcome.
