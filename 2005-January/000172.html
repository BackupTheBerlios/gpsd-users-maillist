On Fri, 31 Dec 2004, Rob McGee wrote:

> It seems like the gpsd mapping clients I've seen work primarily with
> raster graphics. Wouldn't something using vector graphics be better?
>
> I'm just getting into this so I don't know much yet. If there is a
> project which does this, I'd appreciate it if someone would point me to
> it.

you can use xastir with vector maps in shapefile form converted from 
census maps (download them from xastir.tamu.edu). xastir is a gpsd client.

xastir also supports a pile of other things, including its primary purpose 
of being an amateur radio APRS client. but you can use it without even 
touching any amateur radio features.

about all it doesn't support is route-finding; aside from tmrs (which has 
very limited route finding in the code and isn't useful generally yet) i 
know of no open source program which does.
