I am running FC6.

I have got GPSD running perfectly.  If I telnet into port 2947 I get
time and position information (even from a remote machine on my lan).

I even modified the startup of GPSD so it runs as root instead of
nobody, as the gpsd documentation refers to some issue if its not
running as root it cant set the time.  Well running as root makes no
difference at all.

I have put into my ntp.conf file:

# Use GPSD
server 127.127.28.0 minpoll 4 maxpoll 4
fudge 127.127.28.0 time1 0.000 refid GPS
server 127.127.28.1 minpoll 4 maxpoll 4 prefer
fudge 127.127.28.1 refid GPS1

Restarted ntpd.

But ntpq -p reports back I am still using internet time servers, I
don't see my gps listed at all, the only local I see is the standard
fallback that's in ntp.conf

I even commented out the external servers, it still sees no gpsd and
just uses the local fallback.

Any idea's why ntpd refuses to use gpsd?  Obviously gpsd is running
and can provide time.

I do have selinux enabled as its a internet online server.  Do I need
to make a specific rule for gpsd?  The only two rules I have ticked in
selinux is port 22 (ssh) and I made a udp 123 entry for ntpd.

I do have firestarter installed, and have my whole internal subnet
allowed access (hence even my telnet into the server on port 2947 was
from another machine).

All I can think is somehow something in selinux is blocking either the
gpsd from suppling ntpd with time, or ntpd is being blocked from using
gpsd.

Any help appreciated.
