Hi all, I've been using the gpsd packages on gentoo (gpsd-2.32) and
FC5 (gpsd-2.34). I noticed some differences with the gps.py in these -
namely, the Gentoo version seems to have some extra features. For
instance, it has gps.set_thread_hook() (somewhat similar to
gps_set_callback() in the C implementation). I checked the svn
repository and didn't see these features there.

Perhaps these features could be brought into the mainline? (Though I'd
want to make things more consistent first - e.g. set_callback instead
of set_thread_hook, and with gps_set_callback semantics.)

BTW the libgps man page (at least on the website) seem to be out of
date with the implementation (e.g. gps_set_callback's callback
parameter has 2 extra integral args).
