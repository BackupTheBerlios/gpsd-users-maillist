There's been come demand recently for gpsd_config.h to be installed in
/usr/include.  There are several reasons this is a bad idea, beginning
with the fact that we'd effectively lose control over what gpsd is
exporting.

My plan is to make gpsd.h into a generated file that includes definitions
for a subset of the config switches.  To accomplish this, I need to know 
which config switches applications actually need.
