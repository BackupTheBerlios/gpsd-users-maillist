I am using the control socket to send startup commands to a GPS 
receiver. In GPSD 2.96, one could send a command to a receiver by 
sending an exclamation point or an ampersand, followed by the device 
name, the equals sign and a command. For instance, sending the string 
"&/dev/ttyACM0=[command]" worked well for me.

However, sending this string with GPSD 3.1 always returns the string 
"ERROR" on the socket. A little investigation reveals that in the 
handle_control() function located in gpsd.c, this functionality has been 
#ifdef'ed out with the macro __UNUSED__. If I remove the #ifdef, this 
feature seems to work as it did before.

Is there a reason why the ability to directly send commands to a 
receiver has been disabled? Or is there a new way to go about 
accomplishing the same thing? I would prefer not to use a modified 
version of GPSD if at all possible.
