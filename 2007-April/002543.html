Debian will release a new stable version in the very near future. For
the last months it has been encouraged that new upstream package updates
go into the Experimental branch, not Unstable. This is to avoid untested
packages falling automatically into the Testing branch. Now that we are
far enough advanced in the release cycle that the automatic movement of
non-buggy software has been disabled, new packages are again starting to
feed into Unstable, but until Etch is released it still is probably more
polite to put new packages in the Experimental branch.

see the graph, recent jump in the red line indicates the arrival of new
packages in unstable:
  http://bugs.debian.org/release-critical/
