On Fri, 13 Apr 2007, Pete wrote:
> I'm writing up my first libgps app and I guess I don't get how this works
> when using gpsd_poll() with gps_query(gps, "w\n") (where gps is a struct
> gps_data_t*).

I think perhaps the documentation bug I just posted about is biting you.
In your sample code, change

while (1)
   gpsd_poll(GPS);

to

while (1)
   gps_poll(GPS);

(note gps_ instead of gpsd_ ) and see how that works.

The libgps.xml/libgps.3 man page has the correct function name
gps_poll() up top, but the wrong function name gpsd_poll() in the text.

With the sample code you posted, I would maybe expect a compiler
warning; gps_poll and gpsd_poll have different enough signatures that
most newer compilers should have complained a little bit:

int gps_poll (struct gps_data_t *gpsdata);
/* vs */
gps_mask_t gpsd_poll (struct gps_device_t * session);
