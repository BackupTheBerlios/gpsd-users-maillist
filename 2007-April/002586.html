I Have tested a unlisted GPS device that works well with gpsd. I did not 
find any reference to a recipient at gpsd.berlios.de so here it is:

Device: Wintec WBT-201 GPS data logger

1. The GPS model name.
  WBT-201

2. Whether or not it works.
  USBSerial and Bluetooth works

3. If possible, an URL to a technical reference on it.

  Manufacturer Support page with technical specification:
  http://www.wintec.com.tw/en/support_detail.php?cate_id=11&support_id=65

  US Net shop:
  http://www.semsons.com/wiwbmu3gpsre2.html

4. The GPS and (if applicable) USB-to-serial chip it uses. (Look at the 
output of lsusb(1), it may identify the USB-to-serial chip. If it gives 
a vendor and product ID, send us those numbers.)

  Chip set:
  Atmel ATR0625

  dmesg:
  Silicon Labs CP2101/CP2102 RS232 serial adaptor driver v0.07

  usbview:
  CP2102 USB to UART Bridge Controller
  Manufacturer: Silicon Labs
  Serial Number: 0001
  Speed: 12Mb/s (full)
  USB Version:  1.10
  Device Class: 00(>ifc )
  Device Subclass: 00
  Device Protocol: 00
  Maximum Default Endpoint Size: 64
  Number of Configurations: 1
  Vendor Id: 10c4
  Product Id: ea60
  Revision Number:  1.00


5. The interface type (RS232, USB, CF, UART (= 5V TTL RS-232))
  USB + Bluetooth

6. The vendor
  Wintec (Taiwan)

7. Version of gpsd you tested with
  gpsd 2.33 (Debian stable+testing)

8. NMEA version it emits, or '?' if the documentation doesn't say
  Documentation: NMEA 0183 V2.3
