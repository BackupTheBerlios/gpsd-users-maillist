> I have a gpsd and ntpd in connection running on a Fedora 5 box. Sometimes

First, a few more questions:

What version of gpsd are you running?  There have been recent changes
in the PPS logic so try to run the current SVN if you can.

What does 'ntpdc -p' look like when you have a good lock?

> There is a question to GPSD experts, under which circumstances
> GPSD can stop reporting the time to NTPD?

gpsd insists on a valid fix before reporting any time to ntpd.  Depending
on your gpsd version there is also some checking of PPS time relative to
NMEA time.  Be sure to try the SVN for our last best guess at that.
