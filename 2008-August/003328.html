I have been writing some code for an 8-bit embedded system that
syncronises to the NMEA and PPS time coming from a Laipac module.

After debugging my code I couldn't work out why the time was lagging by 1
second (even with PPS).

I plugged the module into GPSD and NTPd to compare just the NMEA time
against some stratum 3 network servers (not PPS) as I wanted to see what
the string offset was in time.  In order to make the module's time lined
up with the stratum 3 references to within a few ms I had to set the time1
fudge from 0.420 to 1.420 - it would appear the module *is* out by one
second!

Has anyone else observed such behaviour from their modules being so out of
whack?  Or better yet is there a fault in my test method above?
