Hello everyone,
The RGM-3800 seems to be working with gpsd. As I did not find that fact
documented either on the hardware page or in the mail list archive, I'll 
assume this is a new finding and report it. I hope that's ok.


GPS model name:               Royaltek RGM-3800
Whether or not it works:      Works (additional tool required)
URL to a technical reference:
http://www.royaltek.com/products_dtl.php?cid=2&id=23&argPage=1&argI=3

GPS and USB-to-serial chip:
lsusb: Bus 002 Device 003: ID 067b:2303 Prolific Technology, Inc. PL2303
Serial Port.
chip (source: reference):  Sirf GSC3f/LP

interface type:               USB
vendor:                       Royaltek
Version of gpsd:              gpsd 2.32 (blame gentoo, not me)
NMEA version (src: reference):NMEA 0183 ver. 3.0
Notes on the device:
This is a GPS data logger with gmouse functionality. Before it can be
used with gpsd, the gmouse functionality has to be switched on. That can
be done with Karsten Petersen's rgm3800py utility. The author of said 
tool has been very helpful to me by describing how to do that on the 
project page: http://code.google.com/p/rgm3800py/wiki/GPSd . Basically, 
one needs to run "rgm3800.py -d /dev/ttyUSB0 gmouse on" before starting 
gpsd.

Attached is a log from the device (when it is put in gmouse mode), as
required by http://gpsd.berlios.de/hardware.html#report_format.

      Philipp Klenze
