I couldn't reproduce this error after configure
--enable-fixed-port-speed=4800, and can't find any identifier 'yes' in
serial.c. 

The last error line is significant, however.  What must be happening
is that configure is stepping on one of the baud rate macros (giving
it the value 'yes').

First thing you should do is svn up.  Then try cc -E and look in the
output rates array to see which one got trashed
