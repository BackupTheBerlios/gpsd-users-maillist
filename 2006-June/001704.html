Marshall Crocker <crocker at cavs.msstate.edu>:
> I'm running gpsd on an embedded system with an STPC Atlas processor
> (486/586 class). I have written an application to connect to gpsd and
> poll every second with the request "p\n".  Whenever I run this
> application, gpsd's CPU usage runs over 70%.  Is there anyway to reduce
> the cpu usage?

I strongly suggest you actually profile a running gpsd and *see* which
bits are spiking your load meter.  No offense, but human beings are so
notoriously bad at intuiting where processor hotspots are that I' not
going to try to do any tuning based on a guess.

As a (possibly) separate issue, I don't recommend polling.  Use watcher
mode instead, through libgps.
