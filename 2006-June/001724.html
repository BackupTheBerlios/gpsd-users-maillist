I noticed that you don't have the Garmin eTrex listed in the hardware
compatability table on the website.

It seems to work OK.

Model: Garmin eTrex ("Vista" model)
Works: Yes
Interface: RS232 (with special connector on the GPS end)
Vendor: Garmin
gpsd version: 2.32
NMEA Version given: ?

gpsprof -f cycle with the interface format (Main Menu->Setup->Interface)
set to NMEA In/NMEA Out:

gpsprof: looking for fix...first fix in 238.45sec, gathering
samples......(270.93 sec) done.
Cycle report Sun Jun 25 15:56:03 2006, Generic NMEA, 4800N1, cycle 1sThe
sentence set emitted by this GPS is: GLL GSV GSA RMC GGA PGRME
GLL: is emitted once a second.
GSV: is emitted once a second.
GSA: is emitted once a second.
RMC: is probably emitted once every 2 seconds.
GGA: is emitted once a second.
PGRME: is emitted once a second.
Send cycle is once per second.


I have also looked at it in minicom and seen HCHDG, GPRMB, GPRTE, GPBOD,
GPVTG, PGRMZ and PGRMM, but I don't know if those are used by gpsd.


With the Interface set to GARMIN, gpsd does not get any data. I suspect
this is their newer proprietary protocol?
