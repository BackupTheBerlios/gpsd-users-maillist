> Sounds like we might have a cgps-specific bug. If you can mail me a
> loh file containin hg several sentences with a nonzero track angle I
> can test for this.
> 
> Actually, you can check it yourself.  Here are the steps:
> 
> 1. Run your log through gpsdecode; check to see if the output JSON has
> a nonzero track angle.  If it doesn't, the daemon is failing.
> 
> 2. Next thing to check is whether xgps shows a nonzeroi track angle.
> Run gpsfake on your logfile to repeatedly present JSON from it on
> port 2947, then run xgps.  Likely it will show there; if it doesn't,
> the Python library binding has a bug.
> 
> 3. Now test the C library binding by running gpxlogger.  If it doesn't
> report the track angle, thesre's a bug in the common C binding.uswd
> by gpxlogger and cgps.
