Hi,

  I am using GPSD with a Garmin 18x through an Serial->USB cable. The
  client is written in C, based on the gpxlogger code example. I
  manually launch the GPSD service because I am using other USB ports
  which have serial port converters and it conflicted with the
  applications using those USB ports.

  Everything works fine except that the track angle always reads zero.
  To see what could be the problem, I started "cgps" and "foxtrot" and
  moved around, always along a straight line towards any of the four
  cardinal points.

  The application "cgps" always gives a zero track angle, but foxtrot
  logs the (approximately) correct track angle. I also checked the raw
  NMEA strings with "gpsmon" and the GPS is (of course) updating the
  track angle as I moved along.
  
  Thus I believe the problem is in reading data from the GPSD. The
  client I wrote is reading the data in the "struct gps_fix_t"
  datastructure. Is there any other way to get the GPS coordinates
  from GPSD ?
