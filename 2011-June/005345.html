I've found some additional information.  Delaying launch of my client by 
a few seconds made no difference.

----------------------------------------------------
But I can now instrument a seemingly similar failure:

If I boot, then remove the gps, then start a client, I get no data (as 
expected).
If I plug in the BU-353 I get no data (unexpected).
If I start a second client, the first client starts returning data 
(unexpected).

What do people think of a watchdog layer for gpsd... if a GPS should be 
returning data but not doing so, have the daemon kick the unit in 
whatever way is necessary?

A watchdog would not only help my embedded unattended application, but 
real-world mapping applications.  How many times have you replugged the 
GPS in a car in order to untangle it from some other passenger's doohickey?

In my embedded world I'm concerned that after a few hours of operation 
the GPS might reset itself (for whatever reason - static - lighting - 
camel spit) and gpsd won't recover.
