Ed W <lists at wildgooses.com>:
> We seem to be talking past each other again. "scons install" is fine
> because this is run on the build machine - it's the *target* that will
> be without python?

I understand that.  In that case, what is a python=no option going to do?
If the target machine doesn't have Python, you can't run scons to begin with.

If all you want is a python=no option that disables Python
installation on the *source* machine, I could do that. But if I do,
occasionally people who want the full install are going to
accidentally set it wrong, and I'm doubtful the very minor convenience
for binary packagers is worth the gripes I'll get about that.
