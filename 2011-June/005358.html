> Now a quick scan of my chroot compile environment shows python2.6
> consuming approx 36MB of disk (excluding headers, ie just lib).  Note
> this may be slightly high because it includes some dependent modules
> which can perhaps be stripped. However, my main distribution is
> currently around 10MB after squashfs, so adding Python will

If you pull git head, you'll notice that the hotplug system now
consists of a sh script (gpsd.hotplug) to be invoked by udev rules,
and a C binary (gpsdctl) which it calls to communicate via sockets
with the gpsd instance.  The old Python hotplug handler is gone.

>                    ...Actually it would be great if the ./configure
> script would re-acquire it's --disable-python option to avoid shipping
> any python deps..?

That I can't do. The build process requires Python in several different
ways.  However, it is not required on the gpsd target machine.
