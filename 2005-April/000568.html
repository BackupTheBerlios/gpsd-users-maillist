Gpsfeed+ is a utility that feeds the PC with continuous GPS data for the
purpose of testing gps applications. NMEA 0183 GPS sentences are sent to
the client application through a socket (TCP/IP) connection or a UDP
(multicast option) message. The same NMEA data can also be sent to a
serial port of the PC (as output). 

gpsfeed+ is free software, written in tcl. It is provided in source
form, licensed under a BSD license. Linux and Windows binaries are also
provided.

The NMEA configuration panel includes the option "User defined",
together with a text box. If this option is selected, whatever is
entered in the textbox, gets output to the ports. Preceding a word with
a  '$' makes it a variable. If the system knows the variable entered, it
will substitute it before it sends the sentence to the output. Otherwise
it will consider the sentence in error and will not output anything from
that sentence.  System variables include NMEA time and date, lat,lon
(NMEA decimal and dms format), epoch time, speed, bearing, satellite
visibility and signal and more.

gpsfeed+ can simulate the GPSTVP sentence proposed by the gpsd project
very easily and it includes it as a user defined sentence example from
version 0.53 on.

Coordinates, altitude and other GPS data output by gpsfeed+  are _fake_.
Although satellite signal reception data is output, this is _not_ based
on any GPS satellite orbit calculation or anything like that. The GPS
data provided by gpsfeed+ are calculated by very simple mathematical
formulas and the results are just presented in the NMEA 0183 sentence
format.

Gpsfeed+ could be configured to allow user definable latency timing,
although this is not currenty implemented in the code.

More info at

http://users.hol.gr/~dzach/gpsfeed/index.html
