We're not home free wrt using casts to structs and dereferencing it's
fields.

Although the use of pragma pack(1) may allow you to fetch a 16-bit value
from unaligned memory by causing the compiler to generate code to do two
fetches, if the 16-bit value is produced in an endianness other than the
processor's, the bytes will be swapped from those intended.  The
compiler can't pretend to know the endianness of the data, only it's
layout.  You'll get the wrong value.
