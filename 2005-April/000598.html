> There doesn't seem to be the sources there so i copied my version of the

There is never a need to move around docbook files.  You just need
to have your xml catalog setup properly.  On Linux the file you
need is /etc/xml/catalog.  Not sure where it is on OS X.

In the catalog file you need a line to tell your xml where to find the
local docbook files that match the ones needed.  Mine looks like this:

  <public publicId="-//OASIS//DTD DocBook XML V4.1.2//EN" uri="file:///usr/share/xml/docbook/dtd/4.1.2/docbookx.dtd"/>

That said, it is slower, but acceptable to get them from the URL your
system was trying:
http://docbook.sourceforge.net/release/xsl/current/manpages/docbook.xsl

Since that failed there is a problem with your xsl installation.  That
is outside of the gpsd project and a pain to debug.

The easiest thing is to just get the man pages from the regular gpsd distro.
