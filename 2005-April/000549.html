I am working with the following setup:

Computer/Build environment:
Mac 17" 1.33 Powerbook, Mac mini 1.44, & G5 1.8DP, all w/Bluetooth
All Mac OS X 10.3.8, latest updates
gcc version 3.3 20030304 (Apple Computer, Inc. build 1666)
	No gcc optimization or compile flags, except '-g'
	Building from the command line with:
		./configure
		make
		sudo make install
	Builds & installs successfully, without any errors or issues (after  
fixing sirfmon.c).

GPS:
GlobalSat BT-318 Bluetooth GPS:
	http://www.globalsat.com.tw/english/products_detail.php? 
main_id=21&p_id=107
Uses Bluetooth Serial Protocol
GPS Protocol (from box):
	Default:  NMEA 0183 (Secondary:  SiRF binary)
	NMEA-0183 v2.2
GPS Output format:
	GGA(1sec), GSA(1sec), GSV(5sec), RMC(1sec), VTG(1sec)
Chipset - Sirf Star II/LP

Software tested with:
gpsd v2.20
gpsd has seg-faulted (see other post)
Palm Pilot Leadtek GPS utility works EVERY time, as NMEA, SiRF & at  
varying communication speeds......

Quick Review/Description:
Quick fix times (experience)
 From box:
	Hot		8 Sec. Avg.
	Warm	38 Sec. Avg.
	Cold		45 Sec. Avg.
	Re-aquire	.1 Sec. Avg.
Bluetooth range is AWSOME!  Box claims 80M!!!!  I've experienced 30'+,  
with a Belkin 100M range Bluetooth adapter.
Comes with Car charger, Home charger, 1650mAh Liion battery, mini CD  
with drivers & NO software.  $159.99 from Semsons.com.
It has a magnetic mount, but is NOT waterproof.  It has a built in  
active antenna & external MMCX antenna connector.  The power button is  
a slider switch.  It can be used without the battery.  There are three  
(3) LED's - Power, Bluetooth connection & GPS Fix.  The charging jack  
looks like an iPaq charging plug.  The battery life is a claimed 15  
hours, in continuous mode, and >20 hours in trickle mode.
Dimensions:  89mm x 52mm x 30mm

I have not had it work with gpsd v2.20, YET.  I have the same  
intermittent problems I had with the Ambicom/Leadtek 9537 Bluetooth GPS  
- Seg-Faults.  It DOES work every time with the Leadtek & SiRFDemo.exe  
programs.....

I still think there's some issue with gpsd, and either the Mac or in  
combination with Bluetooth on the Mac.....  I haven't been able to make  
it reliable since I started using gpsd, with v2.17.  I haven't tried  
any older versions.
