> As a matter of fact the unit was set on the Garmin protocol and that worked
> very well via the serial port with an application called GPSMAN and BTW the
> same application does NOT work anymore with the etrx switched to NMEA :(

There is garmin protocol and there is garmin protocol.  One is
the content of the binary packets and one is the way packets are
transmitted.  YOu can send the binary packets over the RS-232 or USB.
WIth a true Garmin USB the way the pckets are sent is also unique to
Garmin.

The garmin driver in gpsd does not talk garmin over RS-232, or even over
USB.  It can only talk to the linux kernel driver that talks to the
garmin USB device.  The garmin USB protocol uses both the USB control
and data channel in a novel way.  This will not work with serial/USB
converters.

> GPSD seems happy with the eTrex on NMEA.

You are not losing anything but speed.  Unless you have native USB directly
from the GPS you could not get real USB speed anyway.

Have you recompiled kismet to match the version of gpsd you installed?
There are changes to the protocol so the gpsd client and server must match.
