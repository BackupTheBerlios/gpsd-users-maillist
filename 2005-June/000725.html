> To get the new version running I have to crosscompile it. But I have never
> done this before. What I have done now is to set the cross-compiler up. It
> can be run like this :
>        arm-linux-gcc -Wa,-mfpu=fpa11 -mcpu=xscale file.c -o file
try this:

CFLAGS="-Wa,-mfpu=fpa11 -mcpu=xscale" CC="arm-linux-gcc" ./configure

then you can just run `make` to compile the binaries.. you could also
add a specific prefix to the configure script like (./configure
--prefix=/home/foo/ipaq-temp) then you can install it there using `make
install` and finally copy the binaries from there to your Ipaq, this way
you won't have to sort out the compiled binaries..

hope it works and if someone knows a better way, constructive criticism
is always welcome.
