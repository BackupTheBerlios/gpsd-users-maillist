Hi,

I'm having a few problems with a Garmin eTrex gps (the yellow one, 
handheld).

I'm using it with an Acer aspire lappy and using a serial>usb converter 
lead.

I'm running gpsd 2.25 on a suse 9.3 professional installation, and trying to 
use it with kismet as part of a wireless auditing system.

Im in the south west of the UK, and have the garmin set up to use nmea 4800 
baud, setup as:

Position format: hddd mm ss.s
Datum: WGS 84
Units: Statute
North Ref: Magnetic
Variance: 004 w
Angle: Degrees

Take a look at this screenshot:
http://www.telbonic.dsl.pipex.com/kismetgps.jpg

It shows the wierdness quite nicely.  If you look at the kismet output, it 
shows the right speed, but the wrong heading.  If you look at xgps it shows 
the right heading but the wrong speed.  Overlaid on the top is the output 
from gpsd via a telnet session.

When kismet picks up an access point, it shows within seconds as being very 
far away as if im travelling very fast.

Dutch over at the netstumbler forums thought it was an issue with an old 
version of gpsd - now i did have the same problem with the version of gpsd 
that is available after installing gpsdrive via vast.  As it was hinted that 
this was my problem, i uninstalled gpsdrive via yast, then checked to see if 
the command was missing by su to root, then typing "gpsd" to see if i got a 
"missing file" error, which i did.  I then downloaded gpsd 2.25 and 
installed via the rpm.  After a little messing around with a missing library 
i was able to fire up the deamon.

Has anyone else had this problem?
