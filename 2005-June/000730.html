I have a Garmin Etrex Legend C which is a USB only and Garmin binary 
protocol only device. I saw in the changelog that this should now work 
with gpsd, but that it required the kernel module garmin_usb.

I cannot find any trace of such a module on Google. There is a 
garmin_gps module, is that the one? That seems to come from the project 
that also produced the workaround garminusb2nmea. Using garmin_gps and 
garminusb2nmea I made this device talk to gpsd in the past.

My laptop running Debian Sarge finds the device, loads the garmin_gps 
module successfully and attaches it to /dev/ttyUSB0. At least so it says 
in the syslog. Starting gpsd then says

$ gpsd-2.25/gpsd
gpsd: can't run with neither control socket nor devices

Starting it slightly differently produces not a lot of information:

$ ./gpsd -N -D4 -f /dev/ttyUSB0
gpsd: launching (Version 2.25)
gpsd: listening on port gpsd
gpsd: changing to group 20
gpsd: setgid() failed, errno Operation not permitted
gpsd: running with effective group ID 1000
gpsd: running with effective user ID 1000

Am I on the wrong track here?
