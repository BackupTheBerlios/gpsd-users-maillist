Yo Ragnar!

On Fri, 3 Jun 2005, Ragnar Wisloff wrote:

> I have a Garmin Etrex Legend C which is a USB only and Garmin binary protocol
> only device. I saw in the changelog that this should now work with gpsd, but
> that it required the kernel module garmin_usb.

Yes, you need garmin_usb.

> I cannot find any trace of such a module on Google. There is a garmin_gps
> module, is that the one? That seems to come from the project that also
> produced the workaround garminusb2nmea. Using garmin_gps and garminusb2nmea I
> made this device talk to gpsd in the past.

Yes, that is the driver.  It is built in to Linux kernels 2.6.8.1 and
higher.  Just compile it in to your kernel.

> My laptop running Debian Sarge finds the device, loads the garmin_gps module
> successfully and attaches it to /dev/ttyUSB0. At least so it says in the
> syslog.

Sounds good.  If you download hte driver source that you found on Google it
has a simple dump test progeam in it called: garmin_gps_test.  That is a
good independent validation that things are configed properly.

> Starting gpsd then says
>
> $ gpsd-2.25/gpsd
> gpsd: can't run with neither control socket nor devices

You need to start gpsd as root this way:

gpsd -nN -p /dev/ttyUSB0 -D 5

Then sendus the output.
