On Saturday 18 June 2005 01:35 pm, Aart Koelewijn wrote:
> Hello,
> 
> I'm just trying to get gpsd working to use it with kismet and gpsdrive.
> When I try to start gpsd I get the following message:
> Jun 18 20:21:39 localhost gpsd[5857]: gpsd: Can't bind to port gpsd
> Jun 18 20:21:39 localhost gpsd[5857]: gpsd: startup failed, netlib error
> -1
> 
> What can be the cause of this? I'm using Debian Sarge all up to date to
> today, gpsd 2.13. When I use a direct connection from gpsdrive to the
> gps it works ok, but for kismet I need gpsd.

   gpsdrive also uses gpsd.  That error message means you were trying
to start gpsd with an already running instance of the daemon.  The second
process could not bind to the port it needs to listen for clients, so it 
terminated.

   To close the first instance:

kill -SIGINT `pgrep gpsd`

   To see if gpsd is running:

ps aux | grep gpsd
