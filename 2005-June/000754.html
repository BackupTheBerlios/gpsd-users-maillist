Hi:

First of all, I apologize if this is not the appropiate email address to
report "success stories" with gpsd and handheld receivers, to add them to
the list placed at http://gpsd.berlios.de/hardware.html. I looked at the
docs, and this was the only non-individual address to use.

I have just been testing "gpsd" version 2.13-1 (as packaged by Debian Sid)
with a Garmin Geko 201 handheld GPS receiver, and it is working quite nice
so far. What follows is the information needed to update the table in the
project's web page, and additional issues and questions:

1. GPS model name: Garmin Geko 201 (using firmware version 2.50)
http://www.garmin.com/products/geko201/

2. Whether or not it works: it works OK using NMEA protocol, and 4800
bauds in the serial port. I have also tried the garmin binary protocol,
serial port at 4800 bauds, but get no data on the serial port (checked
with "minicom"). All tests done with the "demo mode" of the receiver,
because I am indoors, so take into account.

3. URL to technical documentation:
http://www.garmin.com/manuals/Geko201_OwnersManual.pdf
This is the user guide, although it contains some interesting technical
information, for example NMEA protocol version and NMEA standard and
vendor propietary sentences (summarized later).
There is also an official Garmin GPS Interface Specification available, I
understand you are aware of, at:
http://www.garmin.com/support/pdf/iop_spec.pdf
Not sure if this is specially useful for your project, as it seems more
oriented towards waypoint, track and route data exchange that giving
positioning information to the external world.

4. Data interface: as per the documentation, RS-233
   GPS chip: unknown

5. Interface type: RS-232

6. Vendor: Garmin (http://www.garmin.com)

7. Version of "gpsd" tested: 2.13-1 (Debian Sid package)

8. NMEA version: as per the documentation, "NMEA 0183 version 3.0
interface format is supported by the Geko 201"

9. List of NMEA sentences it emits: as per the documentation:
- NMEA 0183 Version 3.0 Approved Sentences:
  GPGGA, GPGLL, GPGSA, GPGSV, GPRMB, GPRMC, GPRTE, GPVTG, GPWPL,GPBOD
- Garmin Proprietary Sentences:
  PGRME (estimated error), PGRMM (map datum), PGRMZ (altitude), PSLIB
  (beacon receiver control)

10. Notes on the device: cute, nice little piece of useful hardware :-)
I have tried "xgps" to test "gpsd" and the communication with the GPS
device, and realized that every field in the user interface seems to show
correct data, except for the "speed", always at zero. Note that I am using
the "demo" feature, that simulates a constant movement at 20 km/h towards
north, but it is fake GPS data. If you need a sample of that data to check 
if this is some kind of bug, please ask.


That's all, hope it helps. Thank you for the software, regards,
