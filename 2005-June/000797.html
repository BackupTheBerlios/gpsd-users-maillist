jim mcdonagh <jim at mcdonagh.com>:
> I've tried the 2.25 debian package and I've built a 2.26 debian package and 
> the hotplug doesn't seem to work for me (with either version).  I get a 
> broken pipe error when running the gpsd.hotplug script.

And no wonder. The hotplug facility has been plagued with bugs.  Part
of the problem turned out to be that the makefile() method of Python
sockets doesn't play nice with non-blocking operation -- I spent days
thinking my C code was at fault over that one.

I think I've finally got all the problems nailed in the SVN head code,
as of about four days ago.  Good version is in SVN head now, or you
can wait a few days for 2.27 release.

I recommend that the Debian package maintainer go to 2.27 when it
comes out. It's the cleanup release after 2.26, which made a 
boatload of big changes.
