On Thu, 24 Mar 2005, Eric S. Raymond wrote:

> Chris Kuethe <ckuethe at ualberta.ca>:
>> Check the version string: I think an "ES" designation indicates that you
>> have
>> an Enhanced Sensitivity (XTrac) build. We might be able to play games with
>> test mode, running tests that are only supported on XTrac.
>
> I suspect XTrac mode is behind Gene Imes's reports of crazy speeds.  I have
> ES firmware and I'm seeing a less extreme version of the same phenomenon.

XTrac does evil things [ ;) ] with the integrators to keep signal lock in
the face of poor signals. If there is bad multipath and we're locked on to
the incorrect signal it's not too hard to imagine all kinds of errors.

> I'm thinking the Static Navigation control message (ID 143) might solve
> this.  I'm not sure wther the static navigation bit is identical to the
> XTrac bit, though.

Kick the bit and see what happens.

I think static navigation just adds some hysteresis to velocity computation.
Vc < k -> Vr=0.
Vc > k' -> Vr = Vc

Where Vc is computed velocity, Vr is the reported velocity and k/k' are
some kind of control knobs (we might be able to set those).
