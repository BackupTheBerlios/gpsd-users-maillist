Sorry about sitting on the 2.16 release so long.  I wanted to ship a
week ago, but my road trip to Austin slowed me down more than I expected.

I view the release of 2.16 as a major step.  I believe it marks the
end of the massive re-architecting of this code that I began last Fall.
While there is more work to be done, most of it is in the category of
cleanup or relatively easy extensions.  In emails to follow, I'll discuss
some unfinished tasks and possible future directions, beginning with
the simplest stuff and finishing with a trio of blue-sky architectural
issues.

Here is the list of issues I consider currently open.  I'll start an email
thread for each one.

* Rate-limiting SiRF sentence 4.

* Hotplug interface problems

* Should sirfmon be part of the release?

* The mess near error modeling

* HAL integration

* DGPS/WAAS/EGNOS/SBAS support

* Multi-session semantics

* Subsecond polling

* Does the GPSD protocol need to be redesigned?

One reason I'm trying to launch this discussion is that I hope
others are ready to take a hand in the evolution of this code. I've
been doing the heavy lifting for months now, and quite enjoying it --
this is a fun project -- but I could use a bit more help than I've 
been getting.
