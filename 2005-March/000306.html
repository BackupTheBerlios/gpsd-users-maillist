The source distribution now includes a diagnostic tool called sirfmon
for watching the packet stream from a SiRF GPS.  I simplified the tool
and cleaned up the interface a lot while I was using it to debug the
SiRF deriver in gpsd.  Here's the documentation from the header comment:

/*
 * SiRF packet monitor, originally by Rob Janssen <pe1chl at amsat.org>.
 * Heavily hacked by Eric S. Raymond for use with the gpsd project.
 *
 * Autobauds.  Takes a SiRF chip in NMEA mode to binary mode, if needed.
 * The autobauding code is fairly primitive and can sometimes fail to
 * sync properly.  If that happens, just kill and restart sirfmon.
 *
 * Not shipped with gpsd, but we keep it around as a diagnostic tool
 * to double-check gpsd's SiRF decoder.
 *
 * Useful commands:
 *	n -- switch device to NMEA at current speed and exit.
 *	b -- change baud rate.
 *	l -- start logging packets to specified file.
 *	s -- send hex bytes to device.
 *	q -- quit, leaving device in binary mode.
 */

SiRF GPSes have a lot of tuning knobs on them that are accessible
through the binary protocol but not the NMEA one.  Considering how
ubiquitous SiRF GPSes are today (over 80% market share) maybe sirfmon
should turn into a tool for twiddling those knobs and actually be
installed in /usr/bin along with gpsd/xgps/xgpsspeed.

Before that can happen, it needs documentation and more interface
tweaks.  Is anyone interested enough to take responsibility for this?
