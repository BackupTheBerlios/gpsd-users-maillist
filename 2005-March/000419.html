> >I'm thinking the Static Navigation control message (ID 143) might solve

> >XTrac bit, though.
> 
> Kick the bit and see what happens.
> 
> I think static navigation just adds some hysteresis to velocity computation.
> Vc < k -> Vr=0.
> Vc > k' -> Vr = Vc
> 
> Where Vc is computed velocity, Vr is the reported velocity and k/k' are
> some kind of control knobs (we might be able to set those).

It's doing something with distance thresholds too. 

Here's what I'm going to do:

sirfmon will be installed in /usr/bin starting with 2.19; I've written 
a manual page for it.  There will be a sirfmon command that can set
or clear this bit.  I will then ask the people reporting crazy speeds to
set it and see if the problem goes away.

I don't want to put setting static navigation in gpsd itself. That's 
a policy decision that the daemon shouldn't be making.
