> 1. gpsd is intended to be zero-configuration, launched by a hotplug
> script  and automatically syncing up with whatever GPSes it finds. 
> Adding a baud rate option would represent a step away from that, back
> towards the bad old world  where users have to do more than just plug
> the GPS into a handy port to make it work.  

for embedded gps applications time-to-fix is critical to keep power use
to minimum. e.g. I've built GPS->Satellite transmitters that run for yr+
on 15 D-cells. What about a /etc/gpsdrc file with which (if it exists) a
"power user"* can specify [Lock!!!] port/mode/baud rate? This file only
exists if you put it there, Aunt Tillie (below) is none the wiser.
See below for "but it scans only once" response.

[*] I wouldn't really call '-p /dev/ttyS1 -s 9600' "power-usage"

 
> 2. I may give gpsd the capability to handle multiple GPSes.  If that
> happens, an -s option on the command line stops making a lot of sense.

Report from the field re. the use of multiple GPSs:

I know (personally) several different folks using multiple GPSs coming
back to a single logger. This has been for attitude observation on
container ships, development aircraft, and america's cup yachts.

I have never heard a suggestion of using multiple cheap GPS to get a
better fix for where you are*. I think it's a case of- you can have a
dozen TV sets all tuned into the same channel at once, but you are still
just watching the same crummy episode of Gilligan's Island. i.e. you can
improve the signal strength & average out the different interpolation
models each receiver uses, but if you have 12-channel GPSs in the same
place you are still seeing all the same satellites in the sky and you
would be better off just using your best one.

[*] maybe it works great. Sounds like an EE MSc project for someone.
shrug.

> Considering time to first  fix and the fact that you usually pay the

mmph. I'm not sold.. <devil's advocate> if the gps powers down between

when not in use, say in limited memory environments.

I'd rather it just obeyed my orders of where the gps should be and
didn't waste any more time/cycles/probing on the problem. Smart when
needed, sure, but obedient when instructed; and as light as possible.
K.I.S.S. works.
