Yo Eric!

It seems to me that chanceing the output of bad data is not good.  For
this one GPS it is a certainty.

It seems to me if gpsd does not know something it should not guess.  Given
how badly peoples system clocks are set using that is little more than a
guess.

It also breaks any "replay" of saved NMEA.  Glider pilots, and others,
save the NMEA as proof of a fligt profile.  Then paly it back, at a
later date.

I would propose this:
        If we see a GPRMC, then save that date for the next GGA.

	If we have no saved date for GGA, then we skip the date part.
