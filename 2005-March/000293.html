Yo Joe!

I see XMLIBEXPORT nowhere in the /usr/X11/include tree on Xorg.

Got a fresh Slackware 10.1 install here, XMLIBEXPORT is used in one
file /usr/X11/include/Xm/WorldP.h, but defined nowhere.

Found this in /usr/X11/LessTif/ReleaseNotes.txt:

     * Fixes by Dave Williss for the (Microsoft) Windows platform. One of
       the  consequences  is  that  the macro XMLIBEXPORT is used in many
       places.  On  the  Windows platform, we configure the build so this
       means something to the compiler; on others it is blank.

So maybe try to add something like this to the file that will not compile:

#define XMLIBEXPORT ""

and generic Intel and AMD mobos. I got audio problems, SMP problems,
USB2 problems, firewire problems...  Dunno why the mainline kernel got,
and stayed, so buggy.  Been makeing my own Linux kernels since around 0.9.
