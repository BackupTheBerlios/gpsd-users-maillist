Gary E. Miller <gem at rellim.com>:
> Once again, if the compiler observes pragma pack then this does not
> apply.  Can someone prove the Zaurus CC is so brain dead it does not
> have a pragma pack?  Can someone find ANY modern compiler that does not
> observer pragma pack in some form?

Well, first, I thought we already had a bug report indicating that zgcc
chokes on #pragma pack.  

And second -- even if all compilers support some equivalent of #pragma
pack, that doesn't mean we're home free.  At least two things could 
still go wrong:

1. We might not know what the equivalent command-line option or pragma
or other black magic is, or just not have applied it.  The only way we
could possibly tell that it *needs* to be applied is when some poor
user gets back garbage data.

2. We don't control the optimization switches in the target
environment.  What if one of them turns off #pragma pack, for example
to do access-time optimization?  The C standard permits this; remember
that pragmas are advisory only.

The bottom line is that relying on #pragma pack is bad practice.  Yes,
I know this is a pain in the butt.  I'd rather use your pack-and-cast method
myself, it does make the code easier to read.  But I've learned by painful
experience that to do so is unwise.
