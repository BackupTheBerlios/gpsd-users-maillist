> Rob said:
>> what these users want is either A) the first one, B) an average.
>
> I agree with Rob here.  Roadster[1] only uses GPSD to figure out where

Really, things get interesting with 3 or more receivers. Using two or more
non-collinear baselines (even if they are ultra-short) allows you to do...
position and attitude determination. Like a non-gyroscopic gyro. Guess what
I'm hacking on.

> If GPSD can calculate a more accurate position using multiple devices,
> maybe that's worth doing.  Simpler: just return the first device that
> has a lock.  Even simpler: ignore the multi-GPS case.

Tough to do. You either need to be moving slowly/not at all, or you need
very precise clock synchronization between each of the receivers. By very
precise clock synchronization, I mean small fractions of a microsecond. A
bit of very ugly math can fudge a few receivers together sometimes, but I
wouldn't trust a plane to that construction...
