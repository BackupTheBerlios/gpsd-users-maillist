The error message generated for 'stty -f /dev/tty.gps ispeed 4800' is:  
"stty: tcsetattr: invalid argument".

I suspect this is because the Bluetooth serial device was defined as an 
"Outgoing" port in the utility.  Remember, the GPS DOES connect & work, 
sometimes.

If I try to define a Bluetooth serial device as an "Incoming" port, I 
can't specify the Bluetooth device or service to connect it to.  It 
expects the device to initiate the connection (which the GPS doesn't).  
I tried creating a Bluetooth serial port "Incoming" device, and it 
still gives the same error with 'stty -f /dev/tty.gpsi ispeed 4800'.  
The "Outgoing" device DOES accept the 'stty -f /dev/tty.gps ospeed 
4800' command, but appears to ignore it, and returns a 9600.

BTW:  I am using "ispeed" & "ospeed" WITHOUT the "-".  There was a 
mistake in my original post.  Also, Mac OS X requires a '-f' instead of 
a '-F' for stty.
