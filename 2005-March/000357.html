There is a new N command for inspecting and setting driver modes.
Currently its only use is to allow you to switch between SiRF binary
mode (N=1) and SiRF NMEA mode (N=0).

I implemented this because we got a bug report suggesting that SiRF 
NMEA may be implementing some sort of track-smoothing that SiRF binary
is not doing.  This command should enable us to determine that.

to get any time to look into MID 4 rate-limiting soon, or should I
not wait on that?
