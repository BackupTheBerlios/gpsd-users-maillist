What, if anything, is the connection between GPSD_API_MAJOR_VERSION
(defined in gps.h) and GPSD_PROTO_MAJOR_VERSION (defined in gpsd.h),
and their *_MINOR_* analogs?

From the names, one might think they are different things.

But I see that gpsd sends out GPSD_PROTO_MAJOR_VERSION (gpsd_json.c.)
GPSD_PROTO_MAJOR_VERSION is defined only in gpsd.h. The man page for
libgps refers only to gps.h, not gpsd.h. It also says specifically:

  Take care to conditionalize your code on the major and minor API
  version symbols in gps.h; ideally, force a compilation failure if
  GPSD_API_MAJOR_VERSION is not a version you recognize. See the GPSD
  project website for more information on the protocol and API
  changes.

So I gather in my client I should test the version provided by libgps
against GPSD_API_MAJOR_VERSION, and I should *not* #include gpsd.h so
I can test against GPSD_PROTO_MAJOR_VERSION.

But GPSD_PROTO_MAJOR_VERSION and GPSD_API_MAJOR_VERSION have two
different values:

--------------------------------------------------
ccurley at white:~/versioned/gpsd$ grep MAJOR_VERSION *.c *.h* *.py xgps xgpsspeed
gpsd_json.c:               GPSD_PROTO_MAJOR_VERSION, GPSD_PROTO_MINOR_VERSION);
test_json.c:    assert_integer("proto_major", gpsdata.version.proto_major, GPSD_PROTO_MAJOR_VERSION);
gpsd.h:#define GPSD_PROTO_MAJOR_VERSION 3       /* bump on incompatible changes */
gpsd.h-tail:#define GPSD_PROTO_MAJOR_VERSION    3       /* bump on incompatible changes */
gps.h:#define GPSD_API_MAJOR_VERSION    4       /* bump on incompatible changes */
ccurley at white:~/versioned/gpsd$ 
--------------------------------------------------

Also, the above grep suggests that none of the gpsd clients
distributed with gpsd actually does make a version test as above. Is
this true?
