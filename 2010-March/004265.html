Hi.

There seems to be a problem with the TNT support for gpsd.

In driver_nmea.c in the #ifdef TNT_ENABLE section (lines 659 onwards) is some
code, that tries to access none existend members of gpsd_data_t. This causes a
compile failure, when building with TNT support.

driver_nmea.c: In function ‘processTNTHTM’:
driver_nmea.c:698: error: ‘struct gps_data_t’ has no member named ‘headingStatus’
driver_nmea.c:700: error: ‘struct gps_data_t’ has no member named ‘pitchStatus’
driver_nmea.c:702: error: ‘struct gps_data_t’ has no member named ‘rollStatus’
driver_nmea.c:704: error: ‘struct gps_data_t’ has no member named ‘horzField’
driver_nmea.c:709: error: ‘struct gps_data_t’ has no member named ‘headingStatus’

It is not built by default and so might have slipped through.

At least in 2.91 and 2.92 the bug is present and in the svn head.
