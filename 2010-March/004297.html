Just for the halibut, I pulled in the anonymous git clone, running as
a non-privileged user. Again, as a non-privileged user, everything
compiled just fine. For the "make install", I used sudo to gain root
privileges, and that appeared to run fine.

I notice, however, that several files end up owned by root:

ccurley at white:~/versioned/gpsd$ find -user root -o -group root
./build/lib.linux-i586-2.6
./build/lib.linux-i586-2.6/gps
./build/lib.linux-i586-2.6/gps/client.so
./build/lib.linux-i586-2.6/gps/__init__.py
./build/lib.linux-i586-2.6/gps/packet.so
./build/lib.linux-i586-2.6/gps/gps.py
./build/lib.linux-i586-2.6/gps/fake.py
./stamp-python-modules
ccurley at white:~/versioned/gpsd$

I'm not wild about having files owned by root in a user directory tree.
Is this necessary? Would it be possible to populate the build tree as
the unprivileged user, and change ownership on copying into place?
