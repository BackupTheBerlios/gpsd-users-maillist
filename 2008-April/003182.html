Yes, that is where we first tested it.  Using cgps you will notice that
NOTHING changes when you press "j".  It ignores the setting and
continues to report stale FIX data on signal disconnect no matter how
many times you press "j".  GPSD 2.37 acts like it is in J=1 mode by
default, and permanently in that mode, even though the man page says
otherwise.

I don't think you should trust documentation in any case.  It can be
wrong and often is.  The best thing to do is test.

Thanks for you help.

Sorry, can't check it right now... 
One thing you should know, is that the J option is
handled separately for each gpsd client, so if you
telnet in one terminal window to gpsd, and try to
change the setting while cgps is running in another 
terminal window, it will not affect cgps because it is
another gpsd client.
One thing I can think of, try to press the j key
in cgps. According to the manpage, it switches the
buffering policy on the fly... but it is not likely to
help you because the default setting for gpsd is no
buffering... so I am pretty clueless.

Good luck anyway.
