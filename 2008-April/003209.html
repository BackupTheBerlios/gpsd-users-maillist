I started using gpsd a few days ago with a cheap bluetooth GPS.

I get the following log:

gpsd: opening GPS data source at '/dev/rfcomm0'
gpsd: speed 9600, 8N1
gpsd: => GPS: $PASHQ,RID*28\x0d
gpsd: => GPS: 0299661c0800040200001203 FAILED
gpsd: Navcom: sent command 0x1c (Test Support Block)
gpsd: Navcom: command 0x1c mode = 02, length = 0
gpsd: => GPS: 029966200e000001ae02000071000000f203

rate 00
gpsd: => GPS: 029966200e00000186020a0071000000d003 FAILED
gpsd: Navcom: sent command 0x20 (Data Request) - data block id = 86 at

gpsd: unknown sentence: "$PMTK010,001*2E\x0d\x0a"
gpsd: closing GPS=/dev/rfcomm0 (4)

I think the PMTK might be a MTK extension command as defined in
http://www.sparkfun.com/datasheets/GPS/EB-230-Data-Sheet-V1.2.pdf 
(specially since sending a $PMTK000*32\r\n gets a $PMTK001,101,0*33\r\n 
response).

I'm not familiar with GPS hardware, but should gpsd support or at least 
recognize this extensions to figure out it should use a standard NMEA 
driver? (intead of testing for all the others NMEA subtypes?).

Anyway, gpsd seems to works ok with the device  since it defaults to 
NMEA (it sends the rest of the probes when connecting to gpsd and 
reopening the device).
