I have gpsd running on a Linux 2.4.31 kernel with a PPSkit patch,
against a GPS receiver on a serial port.

 

Using the shared memory driver in my ntp.conf file, exactly as specified
in the man page for gpsd, I have ntpd running, and synchronizing time.

 

So far, so good.

 

I have the PPS signal coming with 1 second intervals on pin 1 (Carrier
Detect) on the serial port (I checked this with a scope). My problem is
that gpsd does not seem to detect the PPS pulse. There are no messages
about PPS pulse detection in /var/log/messages even though I run gpsd
with "-D 5" debug option, and "ntpq -p" shows jitter of 4000 for unit 1.
Memory share unit 0 shows low jitter, and seems to work fine for
synchronizing time, but I need better accuracy, so I have to get the PPS
detection to work.

 

How can I debug this?
