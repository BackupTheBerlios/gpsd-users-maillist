Hi Hamish.

Cannot use a MAX232 in this application, as the Garmin 18 LVC is 5V RS232.

The MAX232 takes a 5V TTL serial inputs (0 & 5V levels) and outputs proper 
RS232 levels (approx +/- 10V, depending on how the RS232 lines are 
loaded.). There is also an inversion in the MAX232.

In this case I need to translate from 5V RS232 to 3.3V Serial - which is a 
level shift (5V <-> 3.3V) and an inversion (RS232 <-> Serial)

I think the big problem here is a terminology one - we're all doing the 
same thing, using the same words to describe it - but assigning different 
meanings to the words!

Dave Hyland provides good set of terminology on the Gumstix serial wiki page.
