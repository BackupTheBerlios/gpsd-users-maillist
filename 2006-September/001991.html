Gary E. Miller wrote:
>> If 2 packets of 96 bytes are being sent and gpsd is doing 64 byte
>> reads and getting those 2 packets as 3 reads of 64, 64, 64, then the
>> problem is in gpsd.
> 
> No, the driver spec is that to packets of 96 bytes should be sent to
> gpsd (or any other client) this way:
> 	read(64) get 64 - first 64 bytes of packet 1
> 	read(64) get 32 - last 32 bytes of packet 1
> 	read(64) get 64 - first 64 bytes of packet 2
> 	read(64) get 32 - last 32 bytes of packet 2
> 
> Yes it is wierd, but that is the way Garmin speced it.

But this is how garmin specified how the USB communication should work, 
  however, we're going through an extra tty layer, and I'm really not 
sure if we can expect to get the exact same packet sizes that the USB 
driver reads when going through the tty layer.
