Yo Massimo!

On Wed, 6 Sep 2006, Massimo Burcheri wrote:

> I received my TNTC unit.
> Now I can't get it connected to gpsd. The TNTC is set to 9600 and two
> NMEA sets are enabled at 60/min.

That is once a second, right?

> $ /usr/sbin/gpsd -n -N -D2 /dev/ttyS0

Try running debug  up a bit hight, like -D5.

> gpsd: opening GPS data source at '/dev/ttyS0'
> gpsd: speed 9600, 8N1

As you can see here.  gpsd first tries to use the port at its current
settings.  Since the port was already at 9600, that is tried first.  The
odd thing is that nothing was detected.

> Is there a way to set the serial settings directly? Also for other
> devices like my Magellan Platinum gpsd takes some time if serial isn't
> set to 9600.

You can configure and compile to a fixed port speed this way:
	configure --enable-fixed-port-speed=9600

That should not be necessary and will like not help in this case.
