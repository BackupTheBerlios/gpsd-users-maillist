  I just got an GPS USB dongle (Locosys uPod, using Atmel ANTARIS 4
chip), and been excited to try it with gpsd. It uses NMEA 2.3, and
most of the thins works well, except for a few messages at the
startup. Here's the log using gpsd:

# Name: Locosys uPod (GPS USB dongle)
# Chipset: Atmel ANTARIS4 ATR0635
# Cycle-time: 1-second
# Submitted-by: "Gergely Imreh"

#
# mode NMEA 0183 ver2.3

# when running gpsd 2.38svn4691 on localhost:2947
# First response from gpsd of "GPSD,R=1,W=1" omitted
# After these lines, stationary mode - works, not shown
$GPTXT,01,01,02,u-blox ag - www.u-blox.com*50
$GPTXT,01,01,02,ANTARIS ATR062x HW 80040001*26
$GPTXT,01,01,02,ROM CORE       5.00    Jan 09 2006 12:00:00*76
$GPTXT,01,01,02,LIC 1EBF-BD07-E83D-6BE1-0F7A*50
$GPTXT,01,01,02,ANTSUPERV=AC SD OD PDoS *0A
$GPTXT,01,01,02,ANTSTATUS=SHORT*6D
$GPTXT,01,01,01,PGRM inv format*34
$GPTXT,01,01,01,PFEC inv format*2C
$GPTXT,01,01,01,PFST inv format*2D
$GPTXT,01,01,01,PFEC inv format*2C
$GPTXT,01,01,01,PASH inv format*36

Then all the $GPTXT lines throw and "unknown sentence" error.
Also, when first starting gpsd, there's a single line of error:

gpsd: launching (Version 2.38dev)
gpsd: listening on port 2947
gpsd: running with effective group ID 0
gpsd: running with effective user ID 0
gpsd: opening GPS data source at '/dev/ttyACM0'
gpsd: speed 9600, 8N1
gpsd: garmin_gps not active.
gpsd: gpsd_activate(0): opened GPS (5)
gpsd: ntpd_link_activate: 0
gpsd: unknown sentence: "$GPTXT,01,01,01,PASH inv format*36\x0d\x0a"
gpsd: closing GPS=/dev/ttyACM0 (5)

When just checking `cat /dev/ttyACM0`, the "inv format" messages don't
appear, just the first six lines of GPTXT.

I have seen, that there might been some messages on this mailing list
about this, just from years before, so I thought it is worth trying it
again.
