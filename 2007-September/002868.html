If I remember right, you're trying to use a CF GPS card in your laptop.
If the GPS card comes up as a normal serial port, you should be able to
run minicom (a serial terminal program), point it at the serial port,
and see _something_ coming from your GPS card - either ASCII NMEA
strings if your GPS is set for that, or binary junk if it's set to
binary mode.  Either way, you should get some kind of reaction.

If you can't even get bytes from your GPS card in minicom, then you have
a more fundamental problem with your CF/PCMCIA setup or something like
that.  Once you can see data in minicom, try gpsd again - if it still
doesn't work, _then_ start troubleshooting gpsd.
