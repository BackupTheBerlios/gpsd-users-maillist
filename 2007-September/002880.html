I can perfectly understand your point of view on this!

While I have good and bad experience both with Windows and Linux, my job 
forces me to use Windows. As it's a pita (IMHO) to maintain two 
different platforms at home and work, so I'll keep on using Windows at 
home as well (exactly for this reason). However, having good Win32 ports 
of open source software available is IMO doing a pretty good job to 
promote open source software in general!!!
> from a software engineering perspective, unmaintained code is bad. the
> responsible thing would be for me to not commit code that i know will
> languish/rot. as i said previously, if someone actually steps up and
> wishes to maintain win32 native code, i would look on it a lot more
> favourably.
>   
Unmaintained code is not - per se - a bad thing. For example: Keeping 
the work of previous attempts to port win32 stuff might help further 
attempts ...

However, think of this: I don't know how many windows developers simply 
"disappeared" looking at the gpsd homepage reading this: "No, we don't 
support Windows — get a better operating system". Even after working 
more than 10 years in the software industry, I don't know what a "better 
operating system" really is! So in my eyes this *is* a political 
statement and nothing else. In fact this statement drove me away from 
any gpsd related development for more than two years!
> as for pretty diffs .... make it easy for people without win32
> experience to a) not be confused by the abstraction layer and b) not
> mess up the abstraction layer
... as always ;-)
