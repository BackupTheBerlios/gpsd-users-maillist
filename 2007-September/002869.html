On Dienstag 04 September 2007, Xamindar wrote:

> Like the N option maybe?  But for me when I
> use the N option it just sits there and doesn't print anything out in
> the console.  I guess it isn't able to even connect to the gps card.

Are you using the latest SVN version? Did you try starting gpsd like this:

	./gpsd -n -N -D2 /dev/ttyS0 -S 2947

Then it should print something. For me it was easier to use gpsd 
than 'minicom' or 'cat' even if the interface was properly configured like

	stty -F /dev/ttyS0 ispeed 4800 ospeed 4800

Maybe some gps need a wakeup.
