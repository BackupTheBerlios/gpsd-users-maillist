Yes and no.

In principle, you could use MinGW:
a) free (as in speech) solution
b) ANSI C99 support (so no need to tweak for the cause mentioned above - 
but to be honest this is really a minor point IMHO)

As VC is simply *the standard* for Windows development, using MSVC has 
some *real* advantages:
A) Windows developers are familiar with it (no need to setup a different 
- strange - toolchain, don't underestimate this!)
B) get a libgps.lib file, so other third-party applications using MSVC 
can easily link to libgps

I can follow the argument of using cygwin - least effort to get gpsd up 
and running (for the reasons already mentioned).

Using MinGW in comparison will cause much more trouble / confusion than 
it will help spreading gpsd on windows ...
