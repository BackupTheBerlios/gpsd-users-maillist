Yo Michael!

On Fri, 28 Sep 2007, Michael St. Laurent wrote:

> Would it be possible to improve the debugging messages for PPS?  I've
> got the impression that there are a few different reasons that the PPS
> signal might be rejected but that the only error is 'pulse too short'.

Not much can be said when debugging a simple pulse.  If the pulse is too
short it is rejected as likely a glitch.  Recheck the quality of your
signal.  Some PPS are TTL levels and not RS-232 levels.  Be sure the
duty cycle is set properly and that your polarity is correct.

If you have a suggestion please offer it up.
