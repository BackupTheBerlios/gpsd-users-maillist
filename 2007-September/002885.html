Hi Peter,

I've been doing just this.

If you need something a little more complex than Chris Kuthe's example in a 
previous post, have a look at using Expect.

I used Expect (http://expect.nist.gov) to connect a telnet session to the 
local gpsd instance and fire off a couple of 'O' commands at it, then parse 
the results.

Expect is TCL based, so anything that can be done in TCL (i.e. string 
parsing, write to file i/o etc) can be done from within Expect.

WARNING.....

I struck an issue with gpsd issuing stale Position & Date data on the first 
command issued in a telnet session. Not sure if this has been fixed in the 
latest SVN or not - I simply fire off two commands roughly a second apart 
at the beginning of a session and ignore the results of the first one.

cheers, Dave
