Hello,

I recently bought a GPS device, and consequently installed gpsd for
the first time. (cgps/xgps/gpspipe etc).

It all works very well.

I wanted to be able to log all my data whilest driving, and so I
created a separate runlevel on my laptop that only did the following:

/etc/init.d/gpsd start

echo w | nc localhost 2947 | grep "GPSD,O=" | /usr/bin/tee -a /root/gps.total |
/usr/sbin/cronolog -p "15 minutes" "/root/%Y-%m-%d-%H-%M.gps" &

cgps


This works well enough for my uses - cgps allows me to check that gpsd
has a lock ( 2 or 3D), and it does log all the data I want.


However, sometimes cgps displays "OFFLINE", and complains about "no
input data" to the console. Everything seems to stop logging, and
doesn't come back.

I have checked the cabling, and it all seems securely plugged in.

I can't really find any explanation of this - whether it's a gpsd
thing, or a GPS device thing?
If it's my GPS unit ( a cheap one:
http://www.discountgpsdirect.co.uk/globalsat-br304-serial-rs232-receiver-p-64.html
), how can I stop this? Is it possible to make gpsd re-start the GPS
device when it detects this?
It's quite awkward to reboot a laptop on a motorway :)

Also, just to make it tricky, sometimes I can go for as long as the
battery on my laptop lasts ( 2 - 2.5 hours) without any instances of
this problem - and sometimes it happens 30 seconds after booting it
up.
