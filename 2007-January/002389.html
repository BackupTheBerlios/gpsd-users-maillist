On 1/10/07, Justin Patrin <papercrane at gmail.com> wrote:
> I'm not sure why they're saying that usbfs is no longer needed or
> used. Probably they think they've made some change to libusb which
> makes it obsolete, but not all programs use libusb.

Having used libusb for other things, and found it to be very easy to
use, I'd just as soon make use of it rather than using all sorts of
insane and flaky kernel modules. Personally, I'd add an extra
dependency on libusb to make the garmin driver work everywhere.

I've grumbled and complained before about depending on python and an
xsltproc... and yet I ship a release using python and xsltproc,
because I was told that there are enough machines with this software
installed that it's not a hardship, and because these are ways to
disable them. By the same token I would suggest that libusb is very
common, very useful, and would make gpsd more usable on more
platforms... and can be made optional.

Gary, your thoughts on using libusb instead of depending on the kernel module?
