Forget my question. I found the solution. I forgot that glib or gtk
init functions set locale. Because of this functions like sscanf don't
work as expected. A setlocale(LC_NUMERIC, "C") did it. Perhaps gpsd
should add a fprintf on stderr or a errno that tells the user that the
sscanf in gps_unpack() wasn't successful. Or perhaps a solution that
isn't influenced by the users locale.
