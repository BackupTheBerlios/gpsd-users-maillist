  The new Hotplug infrastructure under linux include the udev (hotplug)
  subsystem and the system-DBus to notify user land applications.
  AFAIK libusb is replaced by libhal to access the sysfs file system.

  It will be nice if gpsd support this. (As far I know gpsd is currently
  started by a hotplug event).

  *BSD-Guys: How is this solved under OpenBSD/NetBSD? AFAIK the path
  /proc/bus/usb is linux specific too?

Actually /proc is Linux specific.  Some BSDs have /proc, but it is
only about processes.

USB serial adaptors attach and show up as ttys, such as /dev/ttyU0
(and /dev/dtyU0 that ignores CD).

There currently is no equivalent of 'hotplug'.  There should be, and
something that gets attachment notifications from the kernel and sends
a notification that a new device is present would be sensible.
