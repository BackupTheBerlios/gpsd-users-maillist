On 1/9/07, Andreas Stricker <andy at knitter.ch> wrote:
> *BSD-Guys: How is this solved under OpenBSD/NetBSD? AFAIK the path
> /proc/bus/usb is linux specific too?

On OpenBSD, I don't even try to handle hotplug serial devices, but...

we have the hotplug(4) system. /dev/hotplug is read by hotplugd(8)
which in turn fires off a script to do stuff based on device type. I
have a script that watches for disk attachment and mounts them
properly - used to attach my ipod, for example.

OpenBSD does have a rudimentary, somewhat-linux-compatible /proc, but
no one really uses it outside of linux-emulation mode. It doesn't
present the full tree, just processes and basic CPU and memory
information - no /proc/bus. By and large, what linux provides through
/proc, OpenBSD gets with sysctl.

Alas, the kernel does not inlude pci/usb/ata product/vendor/serial
number in hotplug messages, so we usually end up parsing this out of
dmesg if we need it. There was a patch on the tech mailing list to at
least get USB VID/PID for more fine-grain control. I haven't closely
reviewed it, but the principle seems sound to me.

I'm not sure I would activate a serial line with hotplug, because none
of the 8 usb-serial devices give my any sort of unique device
identifier - heck, not even windows gets that. At least with a hard
disk I can examine the serial number and the volume label to decide
how to automount this device.
