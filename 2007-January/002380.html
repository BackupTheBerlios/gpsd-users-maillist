Ulrich Voigt wrote:
> gpsd: Can't open /proc/bus/usb/devices
> 
> I think the problem is the "Can't open /proc/bus/usb/devices" line. As I
> understand the /proc/bus/usb directory is no longer supported by current
> libUSB versions. 
> 
> Does anyone know how to fix this? Is this a gpsd problem or a garmin kernel
> module problem?

A gpsd problem, as the garmin driver directly read from this path without
any library between.

The new Hotplug infrastructure under linux include the udev (hotplug)
subsystem and the system-DBus to notify user land applications.
AFAIK libusb is replaced by libhal to access the sysfs file system.

It will be nice if gpsd support this. (As far I know gpsd is currently
started by a hotplug event).

*BSD-Guys: How is this solved under OpenBSD/NetBSD? AFAIK the path
/proc/bus/usb is linux specific too?
