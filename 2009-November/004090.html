I don't think it's the script per se, but the order in which things
occur at boot.

I don't use bluetooth, so I'm guessing. But it might be enough to run
dpkg-reconfigure gpsd and specify /dev/rfcomm as your device. Then add
the line

rfcomm bind rfcomm0

to /etc/rc.local

The ability to specify the device is designed to work with
hotplugged devices, e.g. USB and bluetooth. gpsd will monitor that
device whether it exists or not. When it comes into existence, gpsd
will detect it and use it. What will happen here is that gpsd will fire
up as it normally does, but won't have a device. When rc.local runs,
later on, that (I am guessing here) will cause the device to come into
existence. gpsd will then pick it up and start using it.

Anyone with more bluetooth experience than I have have a better way to
do this?
