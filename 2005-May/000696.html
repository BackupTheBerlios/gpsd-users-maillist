On Sun, 22 May 2005, Ulrich Ruess wrote:

> I did not run ldconfig, but I did run suseconfig since I was using YAST.

Dunno anything about suseconfig.  Never hurts to run ldconfig twice.

> ldconfig -v | less did not even show the subdirectories below /usr/lib/.

If you do not see the libary in the output of "ldconfig -v" then your
dynamic lib loader is not configired properly.  Look on the file
/etc/ld.so.conf.  There should be one line there for each directory
that has dynamic libraries in it.  The exception in the standard places
like /lib and /usr/lib do not nee to be there. "man ldconfig" will tell
you more.
