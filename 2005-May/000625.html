Looking for a few ideas ... I'd like to be able to show my wife on her
X display where I am while I'm driving to and from the nearby outback
towns.  Say one update per minute, with perhaps vehicle speed, shown as
a map with known points.  Retention of historical data in case I have to
be rescued.

So far I've this kind of configuration.

In the dusty wallaby dented automobile;
- Garmin GPS 12XL or FV-18 module via serial port to laptop,
- laptop runs gpsd on Linux,
- CDMA 1x RTT modem connected via USB,
- NAT'd ISP, 50c per half hour, unlimited data, 144kbit/sec,

Then at home;
- NAT'd ISP, unlimited time, 512kbit/sec.
- the usual Linux systems.

Ideas wanted on;

- how to overcome the dual NAT situation, e.g. is there a mirror or
  reflector server that I can use, otherwise I guess packets to some
  third point,

- how to choose a transmission protocol suitable for the dual NAT *and*
  able to recover from extreme connectivity outages (e.g. five to ten
  minutes while in a long valley),

- how to display the data in a meaningful fashion.
