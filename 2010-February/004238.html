Alexander Carver wrote:

>  > Isn't sparc a gcc2 platform? I think gpsd uses a lot of language
>  > features that only exist in more modern compilers. That could explain
>  > some of the compile failures.
> 
> The version of gcc on the system is gcc 2.95.3 so I'll see what I can do 
> about that if anything can be done.  The strange thing is that I 
> compiled this once before about a year ago.


Good news on this part.  OpenBSD does have gcc3 installed but the actual 
name is egcc.  The system gcc remains "gcc" which is why configure 
picked up on it.  Setting CC=egcc switched everything over to gcc3 and I 
was able to compile gpsd-2.39 successfully from the tarball.

I will try to do the traceback on the Python problem and post the 
results.  I was sort of interested in looking at the JSON interface but 
for now I'm happy that I got the near-latest gpsd compiled.
