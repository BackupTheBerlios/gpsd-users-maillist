Ok, after a lot of tweaking, twiddling, guessing and cursing I managed 
to get gpsd and the GPS to see each other in SiRF binary at 19200 baud 
(38400 doesn't work well on this machine) on OpenBSD/sparc.

I can see all the data through gpsmon, I can telnet into gpsd and watch 
the raw data, that part is good (strangely it dumps core when starting 
up but remains running).

So I configured ntpd 4.2.4 to use the GPS for clock control but all I 
get in the logs is:

SHM shmget (unit 0):  Permission denied
configuration of 127.127.28.0 failed
SHM shmget (unit 1):  Permission denied
configuration of 127.127.28.1 failed

ntp.conf has the recommended/documented configuration:

logfile /var/log/ntpd

server 127.127.1.0
fudge  127.127.1.0 stratum 12

server 127.127.28.0 minpoll 4 maxpoll 4
fudge  127.127.28.0 time1 0.420 refid GPS

server 127.127.28.1 minpoll 4 maxpoll 4 prefer
fudge  127.127.28.1 refid PPS

server pool.ntp.org


Neither GPS nor PPS shows up in ntpq -p.  I know I have to have missed 
something, I just don't know what.
