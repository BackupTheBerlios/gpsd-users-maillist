On Sat, 30 Jun 2007, James Buckle wrote:
> I'm trying to use a serially linked etrex - i was wondering if the gpsd
> has to be launched by an auto-detect system, but "locate gpsd" yields
> nothing either.

I don't know about Fedora in particular, but in general, you might try
looking at the "usual suspects", i.e.

ls /usr/bin/gpsd
ls /usr/local/bin/gpsd

or even

find /usr -name "gpsd"

to see if you actually did get the gpsd executable installed.  If Fedora
has an "/opt" directory you might also look under there:

find /opt -name "gpsd"

Finally, you can get desperate...

find / -name "gpsd"

If you don't have any gpsd on your system at all, you might check the
package description again.  Sometimes things like this get split into
"server" and "client" packages (or even "server", "client" and "utils")
and you have to install all of them to get everything.  If you still
can't find it, then you probably need to talk to the Fedora package
maintainer for gpsd.

If you do have gpsd, you should be able to start it manually if you know
which serial port your GPS is plugged into.  If gpsd is in /usr/bin and
you're using /dev/ttyS0 for the serial port, something like

/usr/bin/gpsd /dev/ttyS0

should start gpsd, and then the local clients, at least, should be able
to find it.  Once that works, you can put it in /etc/init.d or /etc/rc.d
or equal to get it loaded at system startup, or possibly integrate it
with whatever hotplug system you have.
