> void gps_handler(struct gps_data_t *sent, char *buf, size_t len, int level)

I think gps_handler (at least according to the libgps manpage shipped
with 2.34)is only supposed to take two arguments:

void gps_handler(struct gps_data_t *sent, char *buf)

but ISTR some recent discussion on the list about this.

> char server[16];
> char port[5];
> strcpy(server, "127.0.0.1\0");
> strcpy(port, "2947\0");

Probably not your main problem, but I'm pretty sure this will try to
put six characters into port.  In C, string constants in double quotes
automagically get a terminating null appended by the compiler.  Refer to
http://cm.bell-labs.com/cm/cs/who/dmr/ctut.pdf and/or
http://cm.bell-labs.com/cm/cs/who/dmr/cman.pdf .

> printf("Opening GPSd....\n");
> GPS = gps_open(server, port);

/* Check to see if this worked before you pass GPS to set_raw_hook() ! */

if(GPS == null)
   {
   perror("gpsd test program");
   exit(1);
   }
