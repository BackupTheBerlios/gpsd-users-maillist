Hello,
 I have been trying to get PPS working properly with gpsd.

I am using an OEM GPS receiver, I believe it is using a SirfStar I
chipset, it's a SandPiper II module. I'm using gpsd version 2.3.4 on
Fedora Core 6.

I have been running gpsd with "gpsd -n -N -D5 /dev/ttyS0".

I had issues with it not detecting PPS pulses after initialization, but
after looking at other posts, I found a creative solution, which was to
change the baud rate on the GPS module to communicate with SiRF at 9600
baud instead of 38,400. That seems to have resolved that issue - and now
gpsd continues to report DCD changes after initialization.

When I run with -D5 I'm now seeing:
gpsd: pps-detect (DCD) on /dev/ttyS0 changed to 1
gpsd: ntpshm_pps: lost PPS lock
gpsd: Raw SiRF packet type 0x12 length 2: 1201
gpsd: OTS 0x12: send indicator = 1
gpsd: pps-detect (DCD) on /dev/ttyS0 changed to 0
gpsd: PPS pulse rejected too short. cycle: 1008017, duration: 100532

I'm kind of confused - because the cyclce seems awfully close to 1 second
- and the duration looks close to 100ms (which is what the receiver docs
say it should output). If it's supposed to be 1hz.... shouldn't it be
working?

Can anyone offer some insight as to what I'm missing? Or what appears to
be wrong with the PPS signal?

Thanks!
Jim
