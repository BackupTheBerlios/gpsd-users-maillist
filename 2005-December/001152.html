Off-topic, but as long as we're using GPS for accurate position, an
accurate value for MPS_TO_KNOTS (accurate to epsilon for long double) is
1.943844492440604752.  1.9437 isn't a good approximation; 1.9438 would
be closer.  The 'units' program says:

$ units
2084 units, 71 prefixes, 32 nonlinear units

You have: meters per second
You want: knots
        * 1.9438445
        / 0.51444444


(Meters in a nautical mile: exactly 1852; seconds in an hour: 3600)
