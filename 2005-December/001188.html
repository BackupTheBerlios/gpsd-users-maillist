On Fri, 23 Dec 2005, mark_2811 wrote:

> I'm running Mandrake Linux LE 2005.  For some reason the build isn't suc
> cessful with the autogen.sh script.  Is autogen.sh necessary since a reg
> ular 'make' seems to work fine.  What does autogen.sh actually do?  When
>  I try to build gpsd with autogen.sh I get the following messages:

autogen.sh creates configure and configure generates tha makefile.  If
you checked out SVN then you need to run all three in order.  If you
downloaded the release tarball then run just this:
	configure && make && make install

> Your automake version  is older than the suggested one, 1.7.6
> Go on at your own risk. :-)

If you downloaed the SVN then you really need to update your automake
cute.

> [mark at localhost gpsd-2.30]$ ./autogen.sh

That says you are not running as root, so any install will fail later
when the binaries are copied over.


This says you are missing MOTIF, so the x based programs will not compile:

> Makefile.am:8: HAVE_MOTIF does not appear in AM_CONDITIONAL

This says you are missing DBUS support, you willlikely not miss that.

> Makefile.am:14: HAVE_DBUS does not appear in AM_CONDITIONAL

> [mark at localhost gpsd-2.30]$

You left out the details of how you built/installed it.

> I intend to run gpsd from a compact flash card that uses a read-only fil
> e system.  When I launch the executable with '/sbin/gpsd /dev/ttyS4' it
> gives the following:

That is not the proper command line syntax.  You should try this way:
	/sbin/gpsd -p /dev/ttyS4

The naked device as an arugment is not supported.

> mkdir: Cannot create directory '/sbin/.libs': Read-only file system
> /sbin/gpsd: cd: 1: can't cd to /home/mark/projects/gpsd-2.30
> /sbin/gpsd: eval: 1: gcc: not found

This does not even remotely look like gpsd.  It looks like a shell script.
Check the file date, is that really the one you think you compiled?
