Ok, time for a update took long because my ISP is acting up tonight. So
basically getting no where I poked around with the /proc/bus/usb/devices and by
my GPS it has "Cls=02(comm.)" and didn't really understand what that was. So I
looked around the kernel and found cdc_acm and it caught my attention because
the way windows uses the gps. First you install the software then it makes a
"modem" on com4 then you use windows apps though com4. 

So I read the documentation on acm and I saw that same line "Cls=02(comm.)" and
it said "The presence of these three lines is important, it means it's an
ACM device". So I built it as a module then rebuilt the kernel, booted in pluged
in the GPS and check dmesg and see "cdc_acm 4-1:1.0: ttyACM0: USB ACM device". 
SoI quickly cat /dev/ttyACM0 and I see a flow of text. Next I do gpsd -p
/dev/ttyACM0 and then launch xgps which seems to work, and gpsdrive works 
great too!

So enabling ACM in the USB Modem section of the kenerel config did it for me. 
I would like to thank you guys for all the help as I could not have done it
without the help, and I also learned a great deal too.

Thanks,
