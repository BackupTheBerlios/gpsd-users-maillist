Linux Redhat FC4, gpsd 2.30, built and installed from source.

I have a Garmin etrex connected to FC4 on a laptop using a Serial-USB
Adaptor (Targus PA088). This setup used to work OK with older versions
of gpsd. Now I've installed gpsd 2.30, the adaptor thinks its a mouse.
In syslog,  get the error messages: 

Feb  3 11:08:08 hare kernel: input: USB HID v1.10 Mouse [MCT Corp.
Serial Mouse Adaptor] on usb-0000:00:1d.3-2
Feb  3 11:08:08 hare hal.hotplug[4264]: DEVPATH is not set (subsystem
input)

usb 5-2: new full speed USB device using uhci_hcd and address 2
usb 5-2: too many configurations: 67, using maximum allowed: 8
usb 5-2: configuration #1 chosen from 8 choices
input: USB HID v1.10 Mouse [MCT Corp. Serial Mouse Adaptor] on
usb-0000:00:1d.3-2

And X11 goes ape, thinking its getting random mouse actions from the
GPS!

I think this is related to the gpsd hotplug stuff in /etc/hotplug/usb. I
removed the gpsd.* hotplug files but it made no difference. Can anyone
help me (a) undo the damage (b) make the gpsd hotplugging work.
