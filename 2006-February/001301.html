> But, when I telnet into the  server the "escape" key doesn't work to
> exit.  I use the work around of `telnet -e . localhost 2947`.

> 2) perl use GPS:gpsd; gpsd.pm
> I'm not a C programmer by any stretch but, I can hold my own with
> perl.
>  I'd be interested in working to put a gpsd.pm interface together.
> 
> 3) gpsd subscriptions
> Not sure if gpsd wants to get into this, but I'd like to see gpsd go
> subscription based.  Applications can subscribe for updates based on
> various parameters.  For example, I put all the logic in my
> gpsd-tracker application that could have been placed in the gpsd for
> all to use.  For starters, application could subscribe on parameters
> like min time, max time, distance since last report, delta from list
> track report.

> http://twiki.davisnetworks.com/bin/view/Main/GpsApplications
