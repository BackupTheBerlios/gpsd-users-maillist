Hi,
a new user for gpsd, thanks to the devs for providing this sw.

I use a cheap, serially attached GPS mouse made in .tw, called GM100N, made by 
ChengHolin Tech. Corp. gpsd recognizes it as a "Generic NMEA" device, works 
out of the box. The doc of the GPS device says that it uses a "NemeriX" 
16-channel chip, when looking at the hardware I can see a "uBlox" chip [1].

Is it of general interest that I document my GPS mouse any more in depth, 
maybe to use it more fully? The docs show several non-NMEA sentences 
like "$PNMRX..." which allow setting and reading various items of the device 
(ephemeris data, almanach, update rates, speed setting etc.).


Regards,
Ekki, Germany



[1]
uBlox chip identification (larger square chip):
	Atmel
	ublox
	ATR0622
	58A50B 0612
	GP0287

smaller chip says:
	Atmel 605
	ATR0601
	2791703

Both chips are located on a small PCB which is in turn soldered to a larger 
PCB which carries a backup battery, the serial interface circuitry and the 
antenna. Pix available.
