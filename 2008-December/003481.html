On Thu, Nov 27, 2008 at 3:07 AM, PRANEET KUMAR.A <apkumar_be at yahoo.co.in> wrote:
> Hello Team,
>
> I'm new to GPSD and need some basic information to get myself started. Can you please answer the following questions?
>  ·  For establishing connection, what does GPSD expects from lower layers (some host s/w)  if it is configured to read data directly from a socket port? Will it act as a server which keeps listening to a socket port waiting for lower layers to connect to it or is it the other way around?

the protocol is documented in the manpage. (http://gpsd.berlios.de/gpsd.html)

gpsd listens on a tcp port (default 2947) and relays data to connected sockets.

> ·  How does GPSD intimate/communicate the/with lower layers (some host s/w) to provide it with NMEA data? I need this information for initialising my host s/w and feeding the data to the socket when GPSD comes and waits on it.

it doesn't initiate communication with software. it waits for software
to connect. if your software needs to communicate with gpsd, you will
have to attempt to connect, and handle connection failure.

> · When and how would GPSD starts execution? Does this execute when some client application starts and request for location data or does it start at system start-up and run in background?

It starts up however you start it up. Some people start it at boot
time, some people run it by hand, some applications look for gpsd and
try to start it...
