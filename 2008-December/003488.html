I'm running gpsd with a serial GPS on a PC-compatible embedded Redhat 
FC8 system.  The system uses the parallel port to control other 
connected hardware.  When gpsd is running, I have no access to the 
parallel port, and fuser reports gpsd as the process sitting on the 
parallel port.  The serial GPS, gpsd, and my gpsd client code are 
otherwise functioning perfectly.

I'm invoking gpsd as follows:
    /usr/sbin/gpsd -n /dev/ttyUSB0 &

Some questions arise:

1. Why does gpsd take over the parallel port, when my GPS device is on 
the serial port?

2. How do I make gpsd stop using the parallel port?

Without resolution to the above, I'm forced to kill gpsd whenever I need 
to modify the parallel port state, then restart gpsd.  While this works, 
it obviously is not an operationally acceptable solution to the problem.

Help?
