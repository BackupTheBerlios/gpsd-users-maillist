> How big a project would this be for a beginner with gpsd?
> Are there well defined APIs at the gpsd and libusb levels,
> with fairly simply algorithms connecting them?

I am not that familiar with libusb, but judging from all the USB devices
it supports it must be a fairly easy library to use.  Once you can talk
to a Garmin USB device with libusb then plugging that into gpsd should
be straightforward.

A garmin USB devices sends PVT (an similar) packets to the host in
fragments.  The protocol is fairly well documented.  Just get libusb to
assemble those packets into a complete PVT packet and pass it back to
the current garmin_gps functions in gpsd.

If someone could do the first part I could help with the gpsd
integration part.
