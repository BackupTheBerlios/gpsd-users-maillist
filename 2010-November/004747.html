> Over the past year, gps_open_r went away, and gps_open became
> re-entrant. I conjecture that you are using older code, where the two
> co-exist. This leads me to think that moving to the latest from the git
> repo might help.
>   

+1 for what Charles said, libgpsmm in the latest git is reentrant using
the new gps_open().

However, you've said that you are using the C interface and it's working
well, why not stay with that?  I don't see why libgpsmm exists at all --
it is such a thin wrapper over the C functions that there's really no
difference other than an extra function-call on the stack and some
wasted CPU cycles.  Use the C functions if you want the bare-bones
functionality, or write a heavier-weight C++ wrapper that does such
things as throw exceptions on error conditions (I have done so but it's
got some messy dependencies, if I can find some time to clean them up I
will kick it out the door).
