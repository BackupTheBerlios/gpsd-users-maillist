On Tue, Nov 2, 2010 at 6:01 PM, David Severt <severtd at softdevsys.com> wrote:
>  I've attached a patch file for the version 2.90 sources that fixes a
> problem with using gpsd and the Trimble Resolution SMT chip set.  The
> problems fixed are:
>
> 1) Satellite info reporting is inconsistent.  TSIP responds to a request for
> satellite info with a series of 0x5c packets.  The number of packets
> reported varies depending on how many satellites are in view.  Any time the
> number of 0x5c packets returned is less than the maximum number ever seen,
> the SATELLITE_SET (SATELLITE_IS) bit is never set and satellite information
> is not sent to the application.  This can result in long periods of time
> with no satellite information reported to the app.
> 2) Satellite info is not cleared between reports (this is only a problem
> when problem 1) is fixed).
> 3) Altitude info is reported incorrectly.  This is because some
> initialization code is skipped incorrectly.  At the beginning of the
> "tsip_event_hook()" function, the statement:
>
> if (event == event_identified && event == event_reactivate)
> should be-
> if (event == event_identified || event == event_reactivate)
>
> If this is not the proper forum for reporting bugs/fixes, please let me
> know.
>

reasonable place to post the patch,  Do you have sample data capture from your
