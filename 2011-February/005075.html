Hi Dmitriv,

A .tar.gz is just a compressed archive containing all the 
files that make up the gpsd application. You extract it and 
use `configure` and `make` to build it. Take a look at the 
file README that's in the distribution for normal build 
instructions.

To generate gpsd shared libs for the target 
"arm-none-linux-gnueabi", we use:

./configure --host=arm-none-linux-gnueabi --enable-shared
make clean
make

This seems to work well for us.
