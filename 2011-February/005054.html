We're using GPSD and NTPD to sync some embedded devices, so it is
possible.

chrony may be a simpler/better solution, but we need very high accuracy
so we're using NTPD and the GPSs PPS (pulse-per-second) to discipline
the clock.

I'd suggest you look at chrony, but if your heart is set on NTP read on.

NTP rejects time sources when they jump around (or disagree with other
sources).
run "gpspipe -r" and confirm that the timestamps are increasing nicely.
(we had a problem with a GPS trying to send too much data at 9600, it
ended up sending sentences out of order which confused GPSD which in
turn annoyed NTPD).
