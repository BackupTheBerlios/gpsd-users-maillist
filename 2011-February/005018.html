I noticed that when reception isn't very good, the number of satellites
goes up and down. However, I was seeing gps_data.satellites_visible go
up but never come down. I believe the attached patch will handle that
issue.

cgps has a problem that may be related. When it sees a new satellite,
it is marked not used, of course. Once the satellite is marked used, it
is never marked unused. xgps is doing it right, changing the satellites
from used to unused, as appropriate. The code in cgps.c update_gps_panel
looks like it should work, so I don't have a patch.
