Compiling gpsd under OSX required the following steps:

1. libtool to be installed.  Probably some other tools, but they were 
installed for me when I installed another package, so I don't know exactly 
what needed to be applied.

2. The attached patch to gpsd.c and libgpsd_core.c.  gpsd.c needs:
#include <netinet/in.h>
and libgpsd_core.c needs:
#include <stdlib.h>

3. I'm working on a patch to the INSTALL documentation for OSX.

Patch
-----
diff -Naur gpsd-2.2/gpsd.c gpsd-2.2-save/gpsd.c
--- gpsd-2.2/gpsd.c Mon Oct 11 08:57:54 2004
+++ gpsd-2.2-save/gpsd.c    Sun Oct 24 20:08:28 2004
@@ -12,6 +12,7 @@
 #include <setjmp.h>
 #include <stdio.h>
 #include <sys/socket.h>
+#include <netinet/in.h>

 #if defined (HAVE_PATH_H)
 #include <paths.h>
diff -Naur gpsd-2.2/libgpsd_core.c gpsd-2.2-save/libgpsd_core.c
--- gpsd-2.2/libgpsd_core.c Tue Oct  5 10:03:52 2004
+++ gpsd-2.2-save/libgpsd_core.c    Sun Oct 24 20:07:10 2004
@@ -4,6 +4,7 @@
  */
 #include "config.h"
 #include <unistd.h>
+#include <stdlib.h>
 #include <stdio.h>
 #include <string.h>
 #include <netdb.h>
