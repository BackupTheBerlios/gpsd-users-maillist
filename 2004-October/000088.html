With version 2.1 gpsd never seems to stop talking to the gps?

This behavior exists with a qpegps client running locally on my
handheld.  As well as with libgps based code I wrote myself running
locally or remotely.  And also just teleting in, sending a "W", followed
by another "W" and quiting the telnet session. 

My example code was roughed out in a day or to but available here if it
helps:
http://www.sr.unh.edu/src/waypt-20041010.tgz

The above source should build two binaries:

The "waypoint" command attempts to get a decent fix from gpsd and print
out the data on stdout.  Some usefull timestamps and progress data is
sent to stderr while you wait for the fix.  This was something I had
wanted but really a proof of concept for the following command.

The "wpdb" command does the same thing as the above but stores the
information into a sqlite db with a name and label.  

Eventually I intend to make a "trackdb" and a "routedb".  Eventually a
"qroutedb" will provide graphical driving directions and updates with
hooks to other applications (like qpegps) for mapping.  I would also
expect the "waypoint data" to be useful for qpegps, and the addressbook
as well.
