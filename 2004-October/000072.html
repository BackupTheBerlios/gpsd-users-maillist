I've been playing around with this "locod" idea, and I have a
proof-of-concept written (in perl).  It emulates gpsd 2.0 behaviour but 
gets it's location information from other sources, presently only a 
location specified in the config file, or a location inferred from the 
system timezone.  The idea is to add a bunch of methods so that one can 
be assured that at least one will always be able to approximate a 
location for you.

Anyway, to a client (via telnet) it seems to behave just like gpsd, and 
conforms to gpsd as specified in the docs, but kismet won't connect
to it and gpsdrive doesn't like the NMEA statements.  Unfortunately I
don't have a GPS device to compare the real gpsd output.
If you have time any feedback would be appreciated!

http://www.gpster.net/locod.html
