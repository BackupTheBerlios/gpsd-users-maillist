> Hm. That'll require a different message format. For now, the O message
> (7DOF PVT solution with 95% confidence intervals) is what we've got.
> It's a good format. We're not dismissing this extra RTK metadata
> outright, but I don't think it has a place in the standard PVT report.

However, one of the ways we may end up extending O is by adding
optional private-use fields to the end of the report.  These would be
used for numeric metadata that is closely coupled to a PVT report --
for example, water depth or temperature.

Those would differ from the separate GPSD-NG queries, which would be
for things like satellite ephemeris data *not* closely associated 
with an individual PVT report.
