It's been a busy six months since our last release, but here we have
it: gpsd 2.34.

In this release we've addressed performance and reliability concerns
and made security enhancements. A few examples:
* improvements have been made to the regression testing framework
* we now have the beginnings of a mechanism to exhaustively compile
test all configure options.
* profiling revealed severe "hot spots" which have been addressed
* several configure options have been added to tune gpsd for embedded use
* efforts are made to detect buggy embedded toolchains
* a number of drivers (evermore, garmin, ntp and tsip) have been
cleaned up and/or made to work with newer receivers
* client applications don't spin and consume all cpu when the server goes away
* more runtime validity checks have been added to prevent gpsd from
crashing on bad data
* gpsd uses no unbounded string functions

Gpsd 2.34 was brought to you by:
Mick Durkin (documentation, bug fixes)
Jeff Francis (cgps, testing)
Chris Kuethe (mad hacking)
Bill Marr (documentation)
Gary Miller (gpspipe, garmin, ntp)
Eric Raymond (mad hacking)
Petr Slansky (evermore driver, documentation, bug fixes)

This is the first release with me wearing the "release engineer" hat -
my apologies if I munged the release files.
