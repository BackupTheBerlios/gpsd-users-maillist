Well, as for Xplanet, I already plot the GPS constellation positions
and the last 30 minutes path. I just have to give xplanet two files:
1. TLE data for the objects I want to track - you can get them from
celestrak.com
2. Configuration for plotting, as the following line demonstrates:
20830 "GPS 15" trail={ground,-20,0,10} color=green

first field is the object ID (20830 is GPS BII-09)
2nd field is a display string
3rd are parameters for displaying path: relative to ground, previous
20 min, next 10 min.
4th: color =] (rgb.txt applies)
(lots of parameters, RTFM as usual)

Then you have to edit xplanet's config file and add a line:
satellite_file=path/to/your/satellite/file

Or else, you could write a markers file with the following syntax:

-22.88  -43.28 "Rio de Janeiro"   color=DarkOrange1 (self documenting =])

and the add a line to xplanet config file:
marker_file=path/to/your/conf/file

But then you need to know lat/long for each sat.

Xplanet has a fairly comprehensive list of projections, including
azimuthal, which would fit the task very nicely.

$ xplanet -config=/path/to/your/config -projection azimuthal -latitude
-22.88 -longitude -43.28 -radius 200 -window_title "Azimutal - Rio"

This plots a nice azimuthal map over Rio de Janeiro. It should be easy
to script this to automagically plot an az map over the gpsd location.

I am willing to test this if someone is willing to implement.
