Chris Kuethe <chris.kuethe at gmail.com>:
> I certainly could use premade icons and import them with
> imagecreatefrompng(), but... it was easier to just use some of the
> built-in drawing functions to slap some dots into an image. They can
> be rescaled easily, and I don't need to have any extra files or huge
> constants (embedding the pixmap) in my program. I also didn't need to
> find any good satellite icons.

Considerations not lightly to be dismissed.  I would likely have
vetoed a PNG-icon-based display this early in the game for a sligtly
different reason -- I don't want us to get invested in big fluffy
icons before we have a clearer idea of what kinds of visual
conventions actually work for this application.  IMNSHO, simple icons in
which every element is informational as opposed to eye candy make a
better place to start exploring from.  Eye candy can be added later.
